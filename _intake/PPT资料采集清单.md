# PPT 资料采集清单（交汇总方）

> **本文件用途**：交给**资料汇总方**（人或负责整理输入材料的 Agent）。请按 **§一** 把资料逐项整理齐、按 **§二** 打包交付；**§三** 是配图真实性的硬要求，**务必知会发起人与采集人**。
> **你不必操心 PPT 怎么排版分页**——那是制作方的活（制作方另有《PPT逐页大纲规范》，与你无关）。你只管把"真实、齐全、标注清楚"的素材交过来。

## 一、资料采集清单（逐项汇总，越全越省事；最少给到 1、2 类即可启动）

### 1. 任务约束（必填，决定整体基调与深浅）
- **汇报对象**：______（领导汇报 / 评审专家 / 客户 / 技术同行 / 其他）
- **场合 + 时长 + 目标页数**：______（如"15 分钟项目评审，约 20 页"）
- **核心目的**：最想让听众记住的 **1–3 个结论** → ______
- **硬要求**：单位 logo / 指定配色 / 保密密级 / 必含内容 / 是否套固定模板 → ______
- **截止时间**：______（精确到几点）

### 2. 内容素材（文本）
- 把全部原始文本（md / word / txt）交来，**无需预先排版**，但请标注：
  - 哪些是**核心**（必上）、哪些是背景 / 附录（可略）
  - 已有的章节 / 目录结构（若有，直接给）

### 3. 真实数据（供制作方画图）
- 凡要呈现的数据，给**真实数值**（表格 / Excel / 文字列数字均可），每组注明：
  - 这组数据**是什么**、**单位**、想强调的**趋势 / 对比**
- 制作方会画成 business-blue 原生**可编辑**图（柱状 / 对比 / 占比 / 表格 / 简单折线）。

### 4. 真实图片（必须真实、不可编造 —— 见 §三）
- 凡"真实结果"——实验曲线、测试结果图、报告 / 系统截图、现场照片、产品 / 设备图——**导出为图片文件**提供。
- **每张图登记一行**（这一步最关键，缺了无法对位）：
  `文件名 | 这是什么 | 建议放在哪部分 / 哪页 | 来源 / 是否可公开`
- 图片尽量原始清晰；带其他系统水印 / 杂边的，标注一下，制作方可裁剪统一。

### 5. 风格与品牌（可选）
- 默认 **business-blue 商务蓝**（制作方已打磨的模板）。
- 要套单位模板 / 指定配色 / 指定字体 → 提供模板文件或色值。

## 二、交付格式（请按此打包）

建议交付**一个文件夹**：

```
/<项目名>_PPT素材/
  00_任务约束.md       # 对应 §一.1（汇报对象/时长/页数/核心结论/硬要求/截止）
  01_正文资料.md        # 对应 §一.2（全部文本，可多个文件）
  02_数据.xlsx 或 .md   # 对应 §一.3（真实数值 + 每组说明）
  03_图片/              # 对应 §一.4
     图片清单.md         # ★ 必备：每张图登记一行（文件名|是什么|放哪|来源）
     <真实图片文件...>
  04_品牌模板/          # 对应 §一.5（可选）
```

> **铁律**：`03_图片/图片清单.md` 必须有，否则制作方无法把图对位到页面。

---

## 三、配图真实性 · 硬边界（务必知会发起人与采集人）

- **真实结果 / 数据 / 图，必须真实提供**——实验曲线、测试结果、现场照片、系统截图、报告数据，制作方**不编造、不 AI 生成、不用假数据凑**。
- **制作方负责**：把真实数据画成专业可编辑图、把真实图片排版美化（裁剪 / 加边框 / 标注 / 底衬）、做装饰 / 图标 / 示意结构图 / 整体版式设计。
- **因此**：制作方出的大纲里凡标 **【需提供】** 的页，必须在 `03_图片/` 或 `02_数据` 里有对应真实素材，否则该页**无法定稿**。汇总时请对照补齐。
