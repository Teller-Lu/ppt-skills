# PPT 逐页大纲规范（制作方工作底稿）

> **本文件用途**：**制作方（我 / 专门制作 PPT 的 Agent）自用**。资料汇总到位后，按本规范把素材**规划成逐页大纲初稿**，交发起人审；审定后才进入制作。
> **不发给汇总方**——汇总方只看《PPT资料采集清单》。两份文件配对：清单管"输入什么"，本规范管"拼成什么结构"。
> **成品形态**：business-blue 商务蓝、可编辑 pptx。

---

## 一、逐页大纲字段规范

资料汇总到位后，输出一份「逐页大纲」初稿给发起人审。每页一个条目：

```
### P<页号> · <页标题>  [content_type]
- 要点：<这页讲的 2–4 条精炼要点(已浓缩、非整段原文)>
- 版式：<见 §二 content_type 清单>
- 配图：<【我画】XX 数据图(注明数据来源) | 【需提供】XX 真实图(对应图片清单第N张) | 【我做】示意/图标/装饰>
- 备注：<强调/过渡/留版面 等，可空>
```

**配图标注三类**（出大纲时逐页标清；真实性硬边界见 §四）：
- **【我画】** —— 用户给的**真实数值**，我画成 business-blue 可编辑图（柱 / 对比 / 占比 / 折线 / 表格）。须注明数据出处。
- **【需提供】** —— 必须用户**真实图片**（实验曲线 / 报告截图 / 现场照片…）。须注明对应"图片清单第 N 张"。**缺则该页无法定稿。**
- **【我做】** —— 装饰 / 图标 / 示意结构图 / 整体版式，由我原生绘制。

---

## 二、content_type 选型清单

据要点性质选版式（我自动选、发起人可改）。库内已覆盖页型：

| 内容性质 | 用哪种版式 | 库内件 |
|---|---|---|
| 封面 / 目录 / 章节过渡 / 结尾致谢 | 对应签名页 | cover / toc / section / closing |
| 2 项并列对比 | 对比（双圆） | seg-2 |
| 3 / 4 / 5 要素并列 | 三段 / 四段 / 多段（枢纽） | seg-3 / 4 / multi |
| 流程 / 阶段推进 | 时间轴 | timeline-snake |
| 逻辑关系 / 系统架构 / 闭环 | 逻辑图 | logic-cycle |
| 数据呈现 | 数据图（柱 / 对比 / 占比） | data-chart |
| 排名 / 明细 | 表格 | table-rank |
| 一段说明 + 配图 | 图文 / 一段文字 | showcase / text-block |
| 多图 / 多模块展示 | 多图排版 | bz-multi-image（4 版式） |

> 同型多风格变体逐步补；详细层叠配方见 `references/页型配方.md`。

## 三、样板页（最终长这样）

```
### P5 · 试验方案总体架构  [逻辑图]
- 要点：三层架构(感知/控制/决策)；各层核心模块；数据闭环流向
- 版式：逻辑图(架构枢纽)
- 配图：【我做】三层架构示意图(原生绘制)
- 备注：承上启下，引出后面各模块详述

### P8 · 关键指标实测结果  [数据图]
- 要点：3 项核心指标全部达标；与基线对比提升 XX%
- 版式：数据图(柱状对比)
- 配图：【我画】指标对比柱状图(数据见 02_数据.md 表2) + 【需提供】台架实测曲线(图片清单 #4)
- 备注：本页是结论重点，留足版面
```

## 四、配图真实性 · 硬边界（制作方自律）

- **不编造任何真实结果**——实验曲线、测试数据、报告截图、现场照片，一律用户真实提供；我**不 AI 生成、不用假数据凑**。
- 我**只**负责：真实数据 → 可编辑图、真实图片 → 排版美化、装饰 / 图标 / 示意 / 版式。
- 凡大纲标 **【需提供】** 而素材未到 → 该页**挂起不定稿**，回头催汇总方按《PPT资料采集清单》补齐。
