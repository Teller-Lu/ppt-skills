# 逐页设计稿 · 范本（四驱扭矩寻优立项 v1.1）

> **这是样板**：以一个真实立项汇报为例，把《逐页设计稿模板》的四问从头填到尾，给新任务照着学"四问怎么填"。
> 立项 PPT 真正定稿后，用**真实终版**回填本范本，保持与成稿一致（活样例，不放空话）。
> 注意它**怎么避开了反模式**：架构 / 计划用 mermaid 不手搓（案例 1、7）、专利对比用表不用 venn（案例 2）、每页一句话主线（案例 5）、价值 / 主张都挂证据（案例 4、6）、不摆空心占位（案例 3）。

---

## 全局块

- **主题 / 标题**：四驱（前后轴）转矩分配寻优 · 立项汇报 v1.1
- **受众**：公司领导 + 评审专家
- **场合**：立项汇报（要的是"认可价值 + 相信可实施 → 通过立项"）
- **时长 / 目标页数**：10 分钟 / ~11 页
- **3 条主线**：
  1. **值得做**——省电 ~1–3% + 弯道能力 +1~3% + 大幅减标定，且有前期证据
  2. **做得成**——意图驱动 + 电压感知的在线寻优，技术路线清晰、平台无关
  3. **难点有解**——车规实时 / 算力 / 维度灾难等每条都有对策
- **风格 / 硬要求**：geely 封面 + 结尾，business-blue 浅色中间页；浅色系；带单位 logo

## 逐页

### P1 封面
- 角色：封面
- ① 一句话：四驱前后轴转矩分配寻优——按意图随工况，在线寻最优。
- ② 表达式：库件（geely 公司风封面）。
- ③ 为什么：封面是"品牌时刻"，正是该用库件的场合（词典：封面≠结构，不走 mermaid）；标题 + 副标 + 单位 logo 一眼定调。
- ④ 真实素材：单位 logo【需提供】（真 logo 不入公开库，实战从 icons_tmp 取）。
- 备注：交 ppt-skills 用 geely 封面版式。

### P2 主线导览
- 角色：过渡（目录）
- ① 一句话：这次汇报就讲三件事——值得做 / 做得成 / 难点有解。
- ② 表达式：并列要点 3 → 库枢纽 / 卡片（词典："并列要点 2–5"是模板强项 ✓）。
- ③ 为什么：3 条主线是并列、对等、各领一段，正合枢纽 / 卡片；不是流程也不是层级，无需 mermaid。
- ④ 真实素材：无（纯版式 + 文字）。
- 备注：三卡分别呼应后面"价值 / 技术 / 难点 + 计划"三段。

### P3 做什么（项目概述）
- 角色：内容
- ① 一句话：让四驱车按意图随工况在线寻优前后轴怎么分，取代固定比例 / 人工查表。
- ② 表达式：流程 / 结构 → **mermaid flowchart**（用源文档 §5 现成那张）。
- ③ 为什么：核心是"踏板→意图→需求转矩 / 多目标→约束→λ→前后轴"这条**信号链**，是流程关系；现成 mermaid 又准又省（词典：结构 / 流程优先 mermaid，**禁手画横向链**＝案例 1）。
- ④ 真实素材：项目介绍 §5 的 mermaid 代码 → 用户用 Mermaid Exporter 导 PNG/SVG。
- 备注：底部一行点出交付边界＝仅前后轴分配。

### P4 价值与收益（评审重点一 · 主线 1）
- 角色：内容
- ① 一句话：省电 ~1–3%、弯道能力 +1~3%、大幅减标定——且都拿得出证据。
- ② 表达式：并列要点 4 → 库卡片，**每卡挂一个证据**（词典：证据 / 成果＝真图 + 标注；并列要点用库）。
- ③ 为什么：四项收益是并列，用卡片；但价值页**最忌无佐证**（案例 4），故每卡配证据数字 / 引用，不空喊。
- ④ 真实素材：电压续航提升 1.78%~2.35% 证据图 `曹开斌_续航对比.png`；节能 / 操纵的 Simulink+CarSim 终版仿真结果【需提供】（§2 自备硬证据）。
- 备注：通用框架一项标"收益但不验收"，别和可量化项混。

### P5 背景 · 驾驶意图现状 → 本项目突破（主线 1 支撑）
- 角色：内容
- ① 一句话：主流走"路况识别→模式离散切换"，通病是切换冲击；本项目用连续意图值做无级自适应。
- ② 表达式：对比（现状 vs 本项目）→ **表格**；佐以企业专利截图（词典：对比维度≥3 用表格，**别套花哨 venn**＝案例 2；证据用真图）。
- ③ 为什么：要讲清"别人怎么做、局限在哪、我强在哪"，是多维对比，表格最清楚；专利截图是可核查证据（案例 6）。
- ④ 真实素材：专利对比表（项目介绍 §3.1 三行）；`专利1北汽.png` / `专利2岚图.png` / `专利3中客.png`。
- 备注：突破点落两句——无级过渡消冲击 + 同一意图值兼作分配基准（一量两用）。

### P6 背景证据 · 经济性电压维度 + 操纵性（主线 1 支撑）
- 角色：内容
- ① 一句话：前期证据已坐实两块红利——电压维度续航 +~2%、无单一型式全工况最优。
- ② 表达式：数据 / 证据 → **真实数据图 + 数据表**（词典：量级 / 数据＝可编辑图表 / 表格，**无真数不编造**）。
- ③ 为什么：这页是"立项的前期证据基础"，靠真实图表说话最硬；E22H 各工况最优型式不同，用数据表呈现（案例 4、6）。
- ④ 真实素材：`曹开斌_电压效率MAP.png`、`曹开斌_续航对比.png`；E22H 工况数据表（§3.3 六行）；可选 `加速过弯.png` / `低附过弯.png`。
- 备注：一句话收口——自适应 λ 逐工况落最优 = best-of-all-worlds。

### P7 技术原理与创新（主线 2 · 评审重点二）
- 角色：内容
- ① 一句话：两大核心——意图驱动（一量两用）+ 电压感知经济性（创新）。
- ② 表达式：核心一的"一个意图值→驱动需求转矩 + 多目标权重"是结构关系 → **mermaid**；核心二配电压效率 MAP 真图。
- ③ 为什么：意图"一处量化、两处驱动"是关系图，mermaid 讲得清（禁手搓框图＝案例 1）；电压创新靠 §3.2 真图证明红利来源。
- ④ 真实素材：意图链可用 `踏板转矩负荷系数.png` / `需求转矩融合.png` / `weight_map.png`（意图×转向角二维权重 MAP）；或我写 mermaid 导图。
- 备注：电压感知点明"二维(T,n)→三维(T,n,U)而不引爆标定量"，对策连到 P9 难点 3。

### P8 技术路线与实现（主线 2）
- 角色：内容
- ① 一句话：七个模块各有确定方案，离线学 + 在线冻结，车规可认证。
- ② 表达式：模块 × 方案 → **表格**（词典：多维并列 / 对应关系用表格）。
- ③ 为什么：7 模块每个对应一套方案 / 策略，是结构化对应关系，表格最紧凑清楚，不需要框图堆砌。
- ④ 真实素材：项目介绍 §5 模块方案表（7 行）。
- 备注：强调"离线学、在线只跑冻结映射"＝车规可认证这条卖点。

### P9 难点与对策（主线 3 · 评审重点三）
- 角色：内容
- ① 一句话：六个真难点，每个都有工程对策——这正是技术含量与立项必要性。
- ② 表达式：难点 → 实质 → 对策，三列 **表格**（词典：多维对应用表格；**别用框图**凑）。
- ③ 为什么：评审最关心"难不难、有没有解"；难点—对策一一对应，表格一目了然，逐条可被追问可被回答。
- ④ 真实素材：项目介绍 §6 难点表（6 行）。
- 备注：开篇先点"难点＝必须跨过的工程障碍，不是价值待证"，区分于 P4。

### P10 里程碑 / 时间线（主线 2 收口）
- 角色：内容
- ① 一句话：2026 H2 启动 → 2027 年中结题，四阶段季度路标。
- ② 表达式：计划 / 排期 → **mermaid gantt（用真实日期）** 或现成 `项目路标图.drawio.svg`（词典：计划用真实日期，**禁手搓网格条 / 编日期**＝案例 7）。
- ③ 为什么：里程碑是带真实日期的排期，mermaid gantt / 现成 drawio 最准；手画必对不齐、日期不实。
- ④ 真实素材：项目介绍 §7 的 mermaid gantt 代码（导图）或 `项目路标图.drawio.svg`。
- 备注：终点标"立项验收 / 结题答辩 2027-06-30"里程碑点。
