# 表达式词典 —— 内容类型 → 最优表达 → 反模式红线

> 四问里 **②表达式 / ③为什么** 的查询手册。**先问"这页内容是什么形态"，再按本表选表达。**
> 最大的坑是**素材优先**：把素材库当锤子、每页都从库里找件套上 → 模板硬套、框图泛滥、手搓结构。本词典就是治这个的。

---

## 三条总纲（每页选表达前先念一遍）

1. **模板是质量地板，不是紧身衣。** 有合适的现成表达才用；**不合适就别套**——宁可更简单（一张表 / 一张大图 / 纯排版列表），也不硬塞不贴的模板。
2. **先定形态、再选表达；框图只是其一，且最易被滥用。** 看到"结构"就画框图、看到"几点"就套卡片 = 没过脑。
3. **结构类（流程 / 层级 / 甘特 / 导图）优先 mermaid 或真实图，手搓原生矢量是最后手段。**

## 主对照表

| 内容类型 | 首选表达 | 次选 | 反模式红线（别默认 / 禁） |
|---|---|---|---|
| **结构 / 架构关系** | 源文档现成 mermaid（导 PNG）> 真实架构截图 | mermaid graph / mindmap 现写 | **🔴 >3 节点的结构禁止手画框图凑**；手搭框图树易碎（踩过） |
| **流程 / 步骤 / 时序** | mermaid flowchart / timeline | 库时间轴件 | 一排框 + 箭头硬画；步骤 >7（拆 / 并） |
| **计划 / 里程碑 / 排期** | mermaid gantt，或用户真实甘特 / drawio | 库时间轴 | **🔴 禁手编日期**——用真实日期；禁手画网格条凑 |
| **对比** | 表格（维度 ≥3 必表格） | 库对比母题 / 双栏 | 硬塞框图；**花哨 venn 慎用**（碍表达就弃） |
| **层级 / 分解** | mermaid mindmap / 金字塔 / 树 | 真实结构图 | 层级没理清就上图；手搭框图树 |
| **量级 / 数据** | 可编辑图表（柱 / 折 / 饼 / 表格） | — | **🔴 无真实数不编造、不画假图**；用框图代替数据 |
| **证据 / 成果** | **真实图 + 标注**（曲线 / 截图 / 照片） | 数据表 | **🔴 主张必配证据**；价值 / 收益页无佐证 = 空 |
| **并列要点 2–5** | **库里枢纽 / 卡片**（这才是模板的强项 ✓） | 好排版列表 | 要点 >5（分组 / 拆页） |
| **单一重点 / 概念** | 一张大图 / 大字 + 留白 | 精心手绘示意图 | 凑框图填满；**装饰性空心图形占位** |
| **罗列 / 说明** | 好排版列表（字号 / 层级 / 留白） | — | 每条套个框；整页无逻辑堆砌 |

> **怎么用**：在四问的 ②写"指到上表哪一类 + 选定表达"，③写"为什么这个最清楚（对照本行红线自证）"。**选不出合适的就降级更简单的，别硬套。**

## mermaid 工作流（结构类首选）

本机已配 Obsidian **Mermaid Exporter** 插件，渲染无忧：
**我写 mermaid 代码 → 用户用插件导成 PNG/SVG → 交 `ppt-skills` 当真实图 place 进 pptx**（可设 mermaid 主题色贴近品牌蓝 azure）。

- **专治**：流程 / 层级 / 架构 / 甘特 / 导图这些**手搓易翻车**的结构图。
- **不全用**：并列枢纽、对比、封面 / 结尾这些"品牌时刻"仍交 `ppt-skills` 用库件——mermaid 图静态、风格通用，按上表分流。

## 逐类细则（红线最硬的几类展开）

### 结构 / 架构关系
- **首选**：源文档若已有 mermaid / drawio / 架构图 → 直接导图用，别重画。
- **何时禁框图**：节点 >3、或有层级 / 分支 → 禁手画横向链 / 框图树，走 mermaid。
- **何时必用 mermaid·真图**：凡"谁连谁 / 谁属于谁"的关系，默认 mermaid 或真实图。

### 计划 / 里程碑
- **必用真实日期**：日期来自项目真实排期，**禁拍脑袋编**。
- **首选**：有现成甘特 / drawio 路标图 → 用真图；否则 mermaid gantt。
- **禁**：手画网格 + 任务条凑一个"看起来像甘特"的东西。

### 量级 / 数据
- **红线**：没有真实数值 → 不画图、不编数。缺数据的页标 **【需提供】**。
- **首选**：真实数 → 可编辑图表（交 `ppt-skills` 画成 business-blue 图）。
- **禁**：用框图 / 示意代替本该是数据图的内容。

### 证据 / 成果（价值 / 收益页尤其）
- **红线**：任何"有价值 / 有效果 / 领先"的主张，**必须配可核查证据**（真实曲线 / 测试图 / 数据 / 文献）。
- **缺证据** → 标 **【需提供】**，绝不用空话 + 装饰图凑一页。

### 概念 / 原理
- **首选**：精心手绘的示意图（讲清机理），而非框图。
- **慎用框图**：框图**只在确为框状结构**（如真有模块边界 / 包含关系）时用；用框图表达"原理 / 流程 / 关系"多半是偷懒。
