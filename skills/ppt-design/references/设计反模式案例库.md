## 案例（v0.1）

### 1. 结构图手画横向链，弃用源文档现成 mermaid
- **症状**：总体架构页，源材料本有 mermaid，却去手画一排框 + 箭头的横向链。
- **为什么错**：手搓结构易碎、对不齐、改一处全乱；现成 mermaid 又准又快。
- **正确做法**：结构关系**优先现成 mermaid / 截图**；没有就 mermaid 现写（导 PNG），手搓矢量是最后手段。
- **出处**：立项 P2。

### 2. 套花哨 venn 模板
- **症状**：为"显得专业"套了花哨的 venn / 三圆交叠模板，但内容并非集合交并。
- **为什么错**：模板是质量地板、不是枷锁；表达式不贴内容，越花哨越碍读。
- **正确做法**：先问内容形态——不是集合关系就别用 venn；花哨碍表达就弃用，改更直白的表达。
- **出处**：立项 P3。

### 3. 摆无信息的空心图形占位
- **症状**：一页摆了 4 个空心图形纯占空间，不承载任何信息。
- **为什么错**：装饰性占位浪费版面、稀释重点，是"凑满"不是"设计"。
- **正确做法**：**每个图形必须承载信息**；没内容就留白 / 并页，禁装饰性占位。
- **出处**：立项 P4。

### 4. 价值 / 收益页无佐证
- **症状**：讲"有价值 / 有收益"，但没有任何数据 / 真图支撑。
- **为什么错**：无证据的价值主张 = 空话，汇报场合最先被专家追问。
- **正确做法**：价值 / 收益主张**必须配证据**（真实数据 / 曲线 / 图）；缺则标【需提供】请用户补。
- **出处**：立项 P5。

### 5. 整页无逻辑堆砌
- **症状**：把相关内容一股脑堆满一页，没有主次、没有一条主线。
- **为什么错**：听众抓不住这页到底要说什么。
- **正确做法**：**每页先定"一句话主线"再排版**；围绕主线取舍，放不下就并 / 拆页。
- **出处**：立项 P6。

### 6. 关键主张缺前沿 / 事实证据
- **症状**：讲技术先进性 / 必要性，缺前沿论文或事实依据。
- **为什么错**：评审场合，无出处的"先进 / 必要"站不住。
- **正确做法**：关键主张配**可核查证据**（文献 / 标准 / 实测）。
- **出处**：立项 P7。

### 7. 计划页手搓里程碑
- **症状**：进度 / 里程碑页手画网格条，日期是拍脑袋编的。
- **为什么错**：手搓甘特对不齐、日期不实，汇报立项尤其忌讳假排期。
- **正确做法**：**用真实日期** + 现成路标 / 甘特图（mermaid gantt 或真实 drawio）。
- **出处**：立项 P10。
