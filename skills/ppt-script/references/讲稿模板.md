# 讲稿模板 —— 讲稿.md 格式 + 写备注 payload 约定

> 两件事：A 给**人审**的 `讲稿.md` 怎么排；B 审完怎么拼成喂 `write_notes.py` 的**备注源**。

---

## A · `讲稿.md`（人审稿）格式

```text
# 讲稿 · {PPT 名}
- 主线：① ____ ② ____ ③ ____
- 目标总时长：____ 分钟

---

## P{n} {页标题}
- 承上：____（把上页连过来 + 锚主线；首页 = 开场白）
- 核心：____（一句大白话，这页就记这个）
- 展开：____（背景 / 为什么 / 举例 / 强调；补充屏幕没说的，别复读 bullet）
- 下页钩子：____（可选，引出下一页）
- ⏱ 约 {X} 秒

## P{n+1} ...
...

---

> 全篇预计：{Y} 分钟（目标 {Z} 分钟）。{若 Y > Z：在此提示"超约 N 分钟，建议精简 P? / P? 的展开"，但不自动砍。}
```

**填写须知**：
1. 四件套里**承上 / 核心**必填，**展开**按页重要性可详可略，**钩子**可选。
2. `⏱ 约 X 秒` 按"该页口语正文字数 ÷ ~200 字/分 × 60"估。
3. 对照《讲稿铁律》逐页自检——尤其**别念稿**（核对成品页文字，讲稿不与之雷同）。

---

## B · 备注源 payload 约定（喂 `write_notes.py`）

`讲稿.md` 经用户审批后，把每页**要进备注的口语正文**拼成一个 txt：

- **口语正文** = 承上 + 核心 + 展开 + 钩子，连成自然的一段话（**不含** `⏱ 约 X 秒` 这类 meta 行、不含 `承上：`/`核心：` 这类标签）。
- **每页之间**用**单独一行** `@@@SLIDE@@@` 分隔。
- **段数必须 = 幻灯片数**（含封面 / 结尾等每一页）。

示例（3 页）：
```text
今天用十分钟讲一件事——四驱车前后轴的扭矩到底怎么分。我们做的，是让它自己在线寻最优。
@@@SLIDE@@@
先说值不值得做。省电和操控的数字在屏上，我重点说为什么省得下来：关键是把母线电压也算进了寻优。
@@@SLIDE@@@
那怎么做？一句话，意图驱动加电压感知的在线寻优。下面拆开讲三块。
```

写入：
```
python skills/ppt-script/scripts/write_notes.py <定稿.pptx> <备注源.txt> -o <带讲稿.pptx>
```
默认输出 `<deck>.noted.pptx`（不覆盖定稿）；段数 ≠ 页数会报错且不写。
