# 个人讲稿风格

> 【**预留 · v1 未实做**】
> v1 用《讲稿铁律》的**通用口语规则打底**，本文是个**接口占位**：将来你愿意提供真实讲稿样本时，按下面的法子蒸出**你个人的口头风格**，覆盖在通用规则之上。现在不实做、生成讲稿时不依赖本文。

---

## 将来怎么蒸（有样本时）

**输入**：你的几份真实材料任一——
- 过往演讲 / 答辩的**讲稿**或**逐字稿**；
- 演讲**录音 / 录像的转写**；
- 你写的、自己讲着顺口的一段话。

**蒸什么**（提炼你的口头习惯，写成几条可套用的风格规则）：
- **口头禅 / 高频连接词**：你习惯用"那""这块""其实""我跟你讲"还是别的；
- **节奏**：偏快密还是慢稳；句子偏短还是中等；
- **详略习惯**：哪类内容你爱展开（举例 / 类比 / 讲故事），哪类一句带过；
- **举例方式**：爱用生活类比、数字对比，还是行业案例；
- **称呼 / 语气**：对领导 / 专家的分寸，谦辞习惯。

**怎么用**：把上面提炼成 `个人风格规则` 数条，生成讲稿时**叠加在《讲稿铁律》之上**——铁律管底线（别念稿 / 口语 / 过渡 / 凝练），个人风格管"听起来像你"。冲突时铁律优先（个人口头禅也不能变成"念屏幕"）。

> 真要启用时，把蒸好的规则写在本文下方，并在 `SKILL.md` 主流程第 2 步加一句"叠加个人风格"。
