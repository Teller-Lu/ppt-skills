# 讲稿铁律 —— 凝练 · 口语化 · 别念稿 · 页间过渡

> 每页讲稿都对照这七条。讲稿是**说给人听的**，不是写给人看的；好讲稿让听众**抬头看你、不低头读屏**。

---

## 铁律一：别念 PPT（最高条）

讲稿**绝不逐字复述屏幕上的字**。屏幕给"**看**的要点"，讲稿给"**说**的展开"——背景、为什么、举例、强调。和页面 bullet 近乎逐字雷同的句子 = 违规。

- **反例**（页面写着"节能 ~1–3%、弯道能力 +1~3%"）：「这一页讲我们的价值，节能百分之一到三，弯道能力提升百分之一到三。」——把屏幕念了一遍，听众读得比你快。
- **正例**：「省电和操控这两块，数字都在屏上了，我重点说说**为什么省得下来**——关键是把母线电压也纳进了寻优，这块以前没人算。」——补屏幕没说的"为什么"。

## 铁律二：一页一个口语核心句

每页开头一句**大白话**点出"这页就让你记住这个"（= 设计稿的 ①一句话核心，翻成口语）。写不出这一句 = 这页你自己还没想清楚要说啥。

- **正例**：「这页就一个意思——**这事值得做，而且我们有证据**。」

## 铁律三：口语化 ≠ 书面语

- **短句、主动、第一/第二人称**（"我们""你看""大家"）。
- **口头连接词**："那""所以""换句话说""这里有意思的是""说白了"。
- **数字 / 术语口语化**：说「提升**大概两个点**」，不念「1.78%~2.35%」；术语**先大白话后学名**——「让车自己判断你是想省电还是想冲，也就是**驾驶意图识别**」。
- **不念**：长定语从句、书面连接词（"综上所述""基于此""由此可见"）、公式符号（λ、Σ 念"分配系数""加权求和"）。
- **默念测试**：出稿前**自己念一遍**，拗口 / 喘不上气就改短。

- **反例**：「基于上述电压效率特性随工况迁移之现象，本项目提出考虑母线电压之轴间转矩分配寻优方法。」——一句书面长句，念出来像念论文。
- **正例**：「我们发现一个现象：**电机最省电的工作点，会随着电压变**。那很自然——把电压也放进来一起算，就能多挤出一点续航。」

## 铁律四：页间过渡 / 逻辑

每页带**承上启下**句，让相邻页有逻辑流，不是孤立 N 段。

- **承上**：一句把上页连过来——「刚才说了**值不值得做**，那到底**怎么做**——」。
- **锚主线**：点明这页属哪条主线、推进到哪（呼应设计稿 3 条主线）。
- **选过渡类型**：按相邻两页关系选 **递进 / 转折 / 因果 / 总分**。
- **反例**：每页一上来就「这一页是技术路线。」——干起，没有和上页的钩连。

## 铁律五：凝练（软标尺）

一页一核心，**不贪多**。按 **~200 字/分**估时长：每页标"约 X 秒"，文末报"全篇预计 Y 分钟 vs 目标 Z 分钟"。**超了只提示、不自动砍**——哪页该详该略由你定（重点页多讲、过渡页一句带过）。

## 铁律六：配图真实性延续

只讲**页面真实素材支持**的内容，不编数据 / 结论 / 出处。页面标了【需提供】还没补的，讲稿对应处也不要替它"圆"出一个说法。

## 铁律七：开场 / 收尾特殊页

- **封面 = 开场白**：今天讲什么、为什么值得你听几分钟（一句话钩住）。不是「这是封面」。
- **结尾 = 收束**：回扣 3 条主线、给一个明确的"所以请支持立项 / 谢谢"，不是「谢谢大家」四个字干收。
