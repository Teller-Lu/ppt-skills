## 三、签名页层叠配方（business-blue 实证，从模板 p8/p17 反推）

> 这是 §1 的具体落地。每层标注"**放哪 · 什么效果 · 用素材库哪个件**"。素材库在 `../assets/business-blue/`（清单 `catalog.json`；下文 `vectors/*.xml`、`rasters/*.png` 均相对此目录）。

### 3.1 封面（p8 实证 · 7 层叠出纵深）
内容文字左半 / 主视觉右半，由下往上叠：
1. **底渐变** 全幅 · periwinkle 柔和渐变 · `vectors/bg-base.xml`
2. **云纹理** 全幅 · duotone（accent1→白）· `rasters/bz-cloud-texture.png`
3. **透明叠加** 全幅 · 压匀色调 · `vectors/bg-overlay.xml`
4. **柔光晕 ×2** · 半透明大径向、半出血 · `vectors/halo-soft.xml`
5. **主标题** 左上 · 渐变大字（烘焙 PNG，原生渐变文字导出被压纯色）· `vectors/title-grad-text.xml`；下接副标（墨色 bold）+ 英文弱灰小字 + 落款胶囊 `vectors/pill-label.xml`（左下）
6. **城市主视觉** 右半吃满 ~1/2 · duotone · `rasters/bz-city-skyline.png`，**略延伸到文字右下方**（凸出节奏）
7. **蓝球 ×2** · 右上塔冠 + 右下 · 受控越界出血 · `vectors/sphere-glow.xml`

### 3.2 目录（p17 实证 · 中心枢纽 + 交错环绕）
1. **底渐变** · `bg-base`（旧 `hex-bg` 幽灵六边已删，其角色并入下面的中心层叠）
2. **CONTENTS** 巨号淡水印字（装饰背景层）
3. **中心枢纽**：**正六边形 + 三层同心"探边"叠**(见 §1.1)做平面立体感——①前：实色蓝块(平缓渐变 + 精调柔阴影)②中：浅蓝垫，比①大一圈，露出像一圈**浅边** ③后：透明带描边六边，再大一圈，露出像一圈**描边环**；"目录"居中反白。**不是** glossy 立体球。模板这三层还整体 3D 旋转(旋转 clone-only，**层叠本身可注入**)。中心真本体素材 = `bz-hex-pad-white`(白透垫层,可注入) + `bz-hex-3d-base`(3D层叠,`scene3d`→clone 或手动粘贴,注入会丢)
4. **6 张目录卡** 白圆角 + 轻投影 + 交错左右环绕（`vectors/toc-card.xml`）
5. **编号六边形** 渐变填充，每卡一个；卡内 `PART X` 眉标 + 中文标题

### 3.3 章节过渡页（暗系 p19 + 亮系 p18 实证）
暗系签名页的"分隔 / 里程碑"感，左文右号：
1. **深蓝满底** + 角落极淡点阵纹理（`bz-dark-bg` + `bz-dot-grid`）。
2. **左侧文字栈**：PART 胶囊（或纯文字）→ **巨大标题（实测 66pt，主文字必须大）** → accent bar → 英文副标。
3. **右侧主角 = 巨大编号**（~1/3 画面），立在等距立方体基座上的 3D 实体。**这块是 `scene3d`/`sp3d` 真 3D，注入会丢 → ③ clone**（等距立方体的明暗也可改 ① 自画三多边形 + 三档蓝，但 3D 厚度数字只能 clone / ④ 烘焙）。
4. 底角 **进度三角 ▸▸▸▸**（`bz-progress-arrows`）点缀。
> **clone 操作法（已验证，p19）**：`Presentation(模板)` → `_sldIdLst` 只留该页 → 按 `shape_id` 删杂件、剥 © / DLP → 用 `slide.shapes.add_*` / 注入素材补我的版式。**新加件与原 `scene3d` 件同文件共存，都能渲**（这就是 clone 能保 3D 的原因）。

**亮系变体（p18 实证 · 0 clone，全 ①自画 + ②图片）** ：与暗系对偶，但更通透 airy。
1. **亮系满底**（`bz-light-bg` 白→淡蓝对角渐变）。
2. **底部宽幅天际线带**（`bz-skyline-band` 全景剪影，②图片；**压成底带、只占小高度**，半透 + 冷蓝薄叠做 recessive）。
3. **左侧文字栈**：PART 胶囊 → **巨大标题 66pt** → 英文副标 14pt（都落在上方通透亮区，避开天际线保对比）。
4. **右侧主角 = 巨大过渡编号**：做到**极大**（实测 430pt 级）、**渐变天蓝**（上浅下深）、**部分藏在天际线楼群之后**——这是设计感来源，也保证编号主导、图不抢戏。
> **喧宾夺主红线（p18 反例纠）**：过渡页的"编号/指示"是主角，**必须最大最突出**；装饰大图（天际线）再好也只能占**小高度带**当衬底。我 v2 的错 = 图占大高度而编号偏小 → 主次颠倒。正解：**编号巨大渐变 + 图压底带 + 编号下沉藏图后**（模板 p18 即此法）。

### 3.4 双栏对比页（p25 实证 · 0 clone，全 ①自画 + 双圆结构素材）
左右对比（传统 vs 智能 / A vs B），中央**同心探边双圆**作母题：
1. **双圆嵌套**（`bz-venn-dual`，可注入）：左 accent1 / 右 accent2，主圆**顶透底实**（线性 alpha 0→~86%）交叠出融合中区；外套大 halo 圆（极淡）+ **渐变光圈细描边的轮廓环**——"同心探边"用到圆上（和目录页六边探边同理）。
2. **圆内**：白线图标（上）+ 24pt 白圆标（下），都**偏外侧**避开交叠区。
3. **要点沿弧分布（关键设计感）🔑**：每条要点配一个小圆点，**均布在轮廓环的圆弧上**（点在弧上、文字从点向外辐射），**不是纵列对齐**——这比规整纵列明显更有设计感（2026-06-18 用户拆模板纠：我 v1 做成纵列就弱了一档）。这是个**通则**：相关小元素（点 / 标签 / 节点）优先**沿母题几何**（圆弧 / 六边 / 曲线）排布，别默认堆成网格 / 纵列。
4. 标题左上 ~30pt 深蓝。
> ⚠ **圆内文字要吃进圆**：圆 / 菱形等非矩形容器**顶底收口、实际可用宽度比外接矩形窄**，文字按圆的真实弦宽收（p25 我的圆标"传统人工运营"溢出了圆边 = 小瑕疵，见 §六 16）。

### 3.5 圆环放射枢纽页（seg-3 三段·p36 实证 · 0 clone，全 ①自画 + ②素材 + 图标）
中心 hub + N 卫星沿环弧分布，适合"三/多段并列"内容（达到模板级，用户认可）。从 p36 反推：
1. **底**：`bz-light-bg` 亮系满底。注入新文件**必先 schemeClr→srgb 归一**（§6.18），否则默认主题 accent2=橙→泛粉。
2. **景深**：2–3 圈极淡 ghost 同心圆（低 alpha 描边、自画），同心于 hub 圆心。
3. **枢纽母题 = `bz-ring-radial`（可注入，本页蒸的）**：azure 渐变粗主环 + **89 根放射刻度条**（仪表盘/雷达感；手画极繁→正该蒸馏复用）+ 内外细描边环；占版 ~45–50%。
4. **中心图标填圆心**：要**够大**（赞踩双图标并排、各 ~0.7"；别在大环里缩成小点留一圈空——2026-06-18 用户：中心可再大）。
5. **N 卫星沿环弧（关键设计感，同 §3.4）🔑**：白圆 + azure 描边 + 轻投影，圆心**落在主环弧上**（三段=10/2/6 点位），内嵌 `templates/icons` 线图标；文字从卫星**向外辐射**（左/右卫星文字右/左对齐贴圆、底卫星居中在下）。
6. **每段文字**：标题（azure 粗，对齐模板实测 22pt）+ 正文（灰 16pt）；**卫星 + 文字别贴页标题太近**（p36 左上"安全意识"段略蹭标题 = 小瑕疵）。
7. **页标题** 左上 ~36pt 粗墨。
> 把 §3.4 的"沿弧分布"从双圆推广到**单 hub + N 卫星**；母题=**圆环刻度盘**（独一份，别和六边 hub/支柱 hub 混）。成品见 `_gallery/seg3-ring-radial`。

### 3.6 六边蜂窝枢纽页（seg-4 四段·p45 实证 · 0 clone，全 ①自画 + ②素材 + 图标）
中心 hub + 4 卫星六边蜂窝镶嵌，适合"四段并列"（达模板级，用户认可）。从 p45 反推：
1. **底**：`bz-light-bg`（注入先 schemeClr→srgb 归一，§6.18）。**别加无意义角落 ghost 装饰**（v1 在左右上角放 ghost 六边、还穿过标题 = 减分，§6.21）。
2. **枢纽母题 = `bz-hex-hub`（可注入，本页蒸的）**：中心 azure 渐变实心六边 + 4 白描边卫星六边（轻投影），**蜂窝偏移**（远端两个同高、内侧两个下沉半身位）；占版充分。
3. **中心图标**：白线图标（link）贴 azure hub——用**透明图标**（见下）免白底方块。
4. **卫星图标**：azure 线图标贴白卫星（`templates/icons`）。
5. **🔑 4 内容卡给内容图形分量（v1→v2 关键，§6.20）**：每段一张**白圆角轻卡**（细描边 + 柔投影），内放 编号（azure 粗 ~30）+ 主标（墨粗 ~20）+ 副（灰 ~14）；远端卡在下、内侧卡在上（随蜂窝错位）。**为什么**：模板把文字做成无背景标签 → 装饰（六边）喧宾夺主；给内容卡后内容与装饰各有图形分量。
6. **页标题** 左上 ~38pt 粗墨。
> **透明图标技法（通用·解决"图标贴深底露白方块"）**：白描边 SVG 渲**黑底** PNG → 取**亮度当 alpha** → 染目标色（`PIL`：`out=Image.new("RGBA",sz,rgb+(0,)); out.putalpha(lum)`）→ 干净抗锯齿的**透明**图标，可贴任意底色（白图标贴深色 hub、彩图标贴白卡都行）。比"白底图标贴白容器"更通用。成品见 `_gallery/seg4-hex-hub`。

### 3.7 五支柱枢纽页（seg-multi 多段·p53 实证 · 0 clone，全 ①自画 + ②素材 + 图标）
中心**承载标题的六边 hub** + 周围 **N 图标圆放射** + **出血外框六边**，适合"多段(5)并列"（达模板级，用户认可，v1 即定稿）。从 p53 反推：
1. **底**：`bz-light-bg`（注入先 schemeClr→srgb 归一，§6.18）。
2. **枢纽母题 = `bz-pillar-hub`（可注入，本页蒸的）**：中心 azure 渐变**尖顶六边**（承载标题）+ 同心**虚线/淡描边外框六边**（出血，蜂窝氛围）+ **5 白圆角图标圆底**（左2右2下1放射，细描边轻投影，含 accent2 小蓝点）。含出血外框 → 注入时 off 对齐画面原点（默认满版）。
3. **🔑 标题进 hub（multi 段省地关键手法）**：段数多(5)，左上角再放页标题会挤 → 把页主标题塞进中心六边（"数字化/五大支柱" 白粗 **28pt** 两行居中，垂直 anchor 居中）。这是 multi 段区别于 seg-3/4（标题在左上角）的手法。
4. **5 段沿放射分布**：每段 = 图标圆底（azure 透明图标，§3.6 技法）+ 标题（azure 粗，模板量得 20→放大 **21pt**）+ 正文（灰，模板 16→**17pt** 够 §1.3 下限）。**对齐向外辐射**：左 2 右对齐（右边缘贴中心图标）、右 2 左对齐、正下居中。
5. **内容图形分量判断（§6.20，与 seg-4 不同·用户认可）**：p53 **不加**文字底板——每段已有**图标圆底当图形锚** + 中心 hub 承载主标题，内容分量「中等偏上」、非裸浮；硬加 5 块底板反破坏放射通透、偏离模板。**与 seg-4（纯六边装饰+裸文字→必须加卡）的关键区别**：当母题本身已给内容锚点，模板文本型式即可，别机械套"加卡"（2026-06-20 用户确认"分析而非盲从"，叮嘱"理念不一定我对，你要分析"）。
> 母题=**支柱枢纽**（中心标题六边 + N 图标圆放射 + 出血外框），独一份，别和圆环刻度盘(seg-3)/六边蜂窝(seg-4)混。成品见 `_gallery/seg-multi-pillar-hub`（亦在合集册 `business-blue-master.pptx`）。

### 3.8 闭环循环图页（logic-diagram·p30 实证 · 0 clone，全 ②素材 + ①自画 + 图标）
左「闭环循环」+ 右「图标步骤说明」双区，适合「流程闭环 / 循环逻辑」（模板级，用户认可）。从 p30 反推：
1. **底**：`bz-light-bg`。
2. **整页母题 = `bz-cycle-loop`（可注入，本页蒸的，左右合一）**——2026-06-20 用户要把原 `bz-cycle-loop`(左) + `bz-step-icons`(右) **合并为一件**（分两件不便设计 / 使用），现整页 logic 母题一件到位：
   - **左闭环**：中心 azure 渐变球 + 4 白节点椭圆 + **4 段弧形循环箭头（顺时针闭环）** + 同心淡环 / 光晕景深。**循环箭头是 cycle 区别于 seg-3 静态刻度盘的关键**（表达「流程闭环」）。**节点 / 中心文字保留 run 槽位**（§0 步4① / §6.22）→ 回填改 `a:t`，位置零偏移、字号自动随模板。
   - **右步骤**：3 个 azure 渐变球错落纵列（含 accent2 小蓝点），每球叠**白色**透明图标——**模板是「蓝球 + 白图标」**（图标 id=38 实测 `lumMod10/lumOff90` ≈白），不是「白块 + 蓝图标」；旁配 小标(azure 粗) + 副标(墨 粗) + 正文(灰)；球数随步骤增删。
   - **合并安全**：左右两区皆 identity 母题（`off==chOff & ext==chExt`）、子件用绝对坐标 → 套一个 identity 父组、bbox 取并集即原样落位，**不触发 §6.22**；左闭环内层缩放组（1.1851×，含保留文字）**保持成组不拍平**（拍平就得重算坐标、又踩坑）。
3. **双区版式**：左图（闭环 ~50%）右文（3 段），还原模板「两区结构」（page-inventory 说的「两区近 seg-2」是指版式，不是两段内容）。
4. **字号**（模板量得）：节点 24 / 中心 24 粗 / 小标·副标 16 / 正文 14 → 正文放大 17（§1.3 下限）；节点 / 中心**保留 run 即自动 24**、不用手设。
> 母题=**闭环 cycle**（中心球 + N 节点 + 循环箭头 + 同心景深）+ 右**图标步骤列**，**整页一件** `bz-cycle-loop`（独一份，别和 seg-3 刻度盘 / seg-4 蜂窝 / seg-multi 支柱混）。成品见 `_gallery/logic-cycle-loop`（亦在合集册）。

### 3.9 蛇形升序时间轴页（timeline·p63 实证 · 0 clone，全 ②素材 + 图标）
