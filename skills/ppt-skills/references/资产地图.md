# 素材资产地图与调用模型

> 设计/生成任何一页**前**,先用本文件搞清:**工作区/skill 里有哪些素材、各自从哪调用、谁参与设计、谁只给人看**。
> 与 `design-language.md §四(出件方式)` 配套——那边讲"每个元素①自己画/②素材/③clone/④烘焙",这边讲"素材具体在哪、怎么取"。

## 一、本风格原生素材库 `assets/<风格>/`(以 business-blue 为例)

| 资源 | 是什么 | 设计时怎么用 | 给谁 |
|---|---|---|---|
| `catalog.json` | **唯一权威清单 / 索引** | 先读它,知道有哪些件 + 每件 `id/file/method/inject/role/effect/place/adjust` | AI |
| `vectors/*.xml` | **矢量源**(原生 DrawingML) | 注入:读 xml → `schemeClr` 上色 → append 到 `spTree` | AI |
| `rasters/*.png` | **图片源**(RGBA / JPG) | 放置:`add_picture(rasters/x.png)`;**PNG 本身可直接 Read 看** | AI |
| 🟢 `builders/*.py` | **参数化 builder 源**(数据驱动件,如表格) | 调 `build_*(slide,x,y,W,headers,rows,…)` 传数据生成;**数据=内容、不写死**(§6.24);`graphicFrame` 非 `grpSp` 故不进 palette | AI |
| `_palette.pptx` | **矢量 + scene3d 的可视化样本册** | 仅两个用途:① 让矢量"显形"(XML 肉眼看不见,渲成样本才好挑)② scene3d 注入即空,**可编辑副本只能存这里 / clone** | AI 看样·clone |
| `_preview/*.png` | `_palette` 每页的渲染缩略图 | 浏览库存效果 | **给人看的** |

### 调用规则(设计/生成时照此取件)
- **矢量件** → 从 `vectors/<file>.xml` **注入**。`_palette` 只是让你"看见"它,**不是调用源**。
- **图片件** → 从 `rasters/<file>.png` 用 `add_picture`。**不从 `_palette` 调**(图片直接可看,不必进 palette)。
- **builder 件**(数据驱动,如表格) → 从 `builders/<file>.py` 调 `build_*()` **传数据生成**(`bz-table-rank` 调 `build_table(slide,x,y,W,headers,rows,bold_cols,band_alpha)`)。`graphicFrame`(表格)非 `grpSp`、进不了 `_palette`/不能 clone → **跳过 palette**,成品看 `_gallery` demo。
- **scene3d 真 3D 件** → 注入必空,只能 **clone**(从 `_palette.pptx` 里已粘好的可编辑副本,或模板原页)。这类件的"可编辑形态"**只存在于 `_palette`**,所以它们必须留在 palette 里。

### 关键约定:catalog(全) ⊋ palette(只矢量 + scene3d)
- `catalog.json` 收录**所有**件(矢量 + 图片 + scene3d + **builder**);`_palette.pptx` 只放**矢量 + scene3d**(图片 / builder 件都跳过)。
- **图片件"在 catalog 有、在 palette 没有"是设计,不是漏同步**——图片从 `rasters/` 调用且本身可直接看,无需进 palette。
- 因此**存素材标准(`蒸馏流程.md §0 步4`)对图片件**:只做"存 `rasters/` + 入 `catalog`",**跳过入 palette / 重渲 preview**;只有**矢量 / scene3d** 才入 `_palette` 并刷 `_preview`。
