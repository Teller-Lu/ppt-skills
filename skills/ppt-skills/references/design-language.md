## 一、设计感 = 四个可操作的来源（别的记不住，先记死这四条）

### 1.0 总纲：平面立体感 ≠ 纯三维（先立这条，再谈下面四条）
**现代（约 2020+）高级感 = 平面立体感**；**纯三维立体感（约 2000–2010s）已过时、显廉价**。这是用户亲授的设计理念（2026-06-17），最高优先。
- **纯三维（别做）**：volumetric 实体、饱和、glossy 高光、强内发光、重棱台——像真实物体（玻璃球 / 拟物按钮）。早期生成的「发光径向渐变球」枢纽就是这种，过时。
- **平面立体感（要做）**：形状本身**保持平面**，深度是**用平面层叠"做出来的幻觉"**——靠 ①**精调柔阴影**（透明度 / 大小 / 模糊 / 角度 / 距离 一起调）②**通透**（半透明填充，玻璃质感但不鼓成球）③**形状边缘叠多重细描边 / 细边**制造层次。模板里每张白色目录卡的"白色透明底"就是这么来的，看上去更高级。
- **几何要究因，别抄表象**：模板中心六边形"看着扁"**不是被压扁**，而是**正六边形 + 3D 旋转**（去掉旋转就是规整正六边形）。所以要画**正多边形**，立体感另用平面层叠（或 3D 旋转）实现，**不是**把形压扁、更**不是**做成 glossy 球。
- **内容归位**：序号 / 文字 / 徽章等功能件**待在所属容器内**；只有装饰才允许凸出（见 §1.4）。
> 一句话：**2020 后的"立体"是平面调出来的层次，不是把东西做成实体。**

### 1.1 景深靠"分层半透明"，不是单层平铺 ← 层次感的根
扁平 = AI 感头号来源。深度不是画出来的，是**多层叠出来的**：每层不同**透明度 + 模糊 + 内容**，光像是穿过去的。
- 一页**背景至少铺到 3 层**：微渐变底 → 柔光氛围圆（半透明径向）→ 幽灵描边环/母题（低透明）。
- 主体之上还可压**前景装饰层**（半透明球/光斑），让画面有"前—中—后"。
- **同心"探边"叠层 = 层次感主力**：同一形状叠 2–3 层、每层比上层**大一圈**，从前到后＝实色块 → 浅色垫(露出来像一圈**浅边**) → 透明带描边(露出来像一圈**描边环**)。模板中心目录块正是这三层(蓝块 / 浅蓝垫 / 透明描边六边)叠出来的——模板还整体 3D 旋转(旋转是 3D 不可注入，但**层叠本身就给足层次**，可注入)。
- 透明用 `fill-opacity`/`stroke-opacity`（SVG，引擎安全）或原生 `<a:alpha>`；**禁** `<mask>`/组 `opacity`/`rgba()`（导出崩，见技法库 D）。
- **反例**：纯色实底矩形直接铺背景 = 立刻显廉价。

### 1.2 光影统一：一个光源、一个方向、一个光色
所有发光与投影必须像来自**同一盏灯**，否则廉价感。
- 投影：所有元素 `dy` **同向**（本系默认光从上来，投影朝下）；`flood-opacity ≤ 0.20`（重了像贴纸，见技法库 T4）；光色用品牌蓝不用灰。
- 发光：球/枢纽用**白色内发光**（~14pt 模糊）模拟高光；暗面用 `lumMod 64%`（亮度 −36%）压暗。
- 一页里**浮起加投影的元素 ≤ 2–3 个**，不是每个块都飘。

### 1.3 占版 · 字号 · 行距：主视觉吃满 ~50%、字号 / 行距按统一刻度
PPT 是给人汇报的；元素小、留白多 = 抓不住重点。
- **主视觉/母题占画面 ~45–55%**，铺满安全区，不龟缩居中（自动门 `min_fill_w/h_ratio` 拦截）。
- 双栏经典分法：文字吃左 1/2，主视觉吃右 1/2（business-blue 封面就是这版式）。
- **🔴 字号刻度**：同一角色跨页**同字体（全库微软雅黑）同字号**、别靠手感定 ——
  - **内容页页标题 = 28pt 粗**、**引导句 / slogan = 28pt 粗**、**正文段落 ≈ 14pt**（普通段落如 p11/p12 必统一；**密集数据 / 表格 / 时间轴说明可豁免**，忠实模板真值）；
  - **签名页主标按模板实测放大**（量 `sz`÷100）：封面主标 ~64–92、章节标题实测 66、过渡大编号 430 级、结尾标题 138 —— **别套内容页的 28**；
  - **硬下限 14**（<14 自动门 FAIL）。一律**先量模板该页真实字号**再落、严禁凭"AI 手感"缩字（见 SKILL §纪律、记忆 [[ppt-readable-scale]]）。
- **🔴 行距 / 段距（统一·别忽松忽紧，2026-06-22 用户加）**：**只对"会折行的多行正文 / 描述段"统一 `lnSpc 120%`**（跨页一致，别有的页 100%、有的 115%），多段之间留**合理段距**。**单行文本不套 lnSpc** —— 标题 / 眉标 / 标签 / 表格单元格 / 签名页大字，单行加 120% 无视觉意义；**数据图表（`bz-bar-chart`）的数值 / 类别精调标签尤其禁套**（`anchor="t"` 单行套 120% 会让基线下移 ~0.02in、破坏像素精调位 —— p13 被全刷的教训）。现状（对）：多行 body 已烤进 `bz-showcase` / `bz-text-block`、多行 desc 已烤进 `bz-team-card` 资产；**其余 builder 文本皆单行 / 精调、本就不带 lnSpc（非遗漏）**。存量回填**只动多行正文页**、**别全量脚本刷**（全刷会污染图表精调标签）。

### 1.4 凸出节奏：绝大多数守边界，极少数（1–2 件）故意越界
"**既有规矩又有凸出**"——这是模板高级感的来源，也是用户亲口点的。
- 内容主体老老实实落在"视觉内容带"里（business-blue：左右≈1/15、上≈1/6、下≈1/9，见 `brand.md`）。
- **只放 1–2 件装饰**（如蓝球）受控冲出安全边出血，制造呼吸感和纵深。
- **凸出只给装饰**：功能性内容（序号 / 文字 / 徽章）必须待在所属容器**内**——把序号做成探出卡片＝错（2026-06-17 用户纠，见 §1.0）。
- 反例：要么全龟缩在框里（呆板），要么全溢出（失控）——都不行。

## 二、让每个"块"不扁平的手段（凡纯色实底块，逐一自问能不能升级）

看到一个纯色填充的块，**生成前就过一遍这张升级表**，能升就升：

| 手段 | 怎么做 | 何时用 |
|---|---|---|
| 渐变填充 | 实色 → `accent→cyan` 招牌渐变 / 径向盘 | 卡片、圆盘、六边、按钮——默认都该渐变，不是纯色 |
| 克制投影 | 加 `flood-opacity ≤0.20` 同向蓝投影 | 需要"浮起"的卡片/CTA（≤2–3 个/页） |
| 半透明叠 | `fill-opacity` 叠一层 | 背景氛围、遮罩、玻璃感面板 |
| ~~内发光/高光~~（慎用） | 白色 innerShdw | ⚠️ 强内发光 = 过时纯三维(glossy)，**少用**；现代要平面立体感(§1.0) |
| 幽灵描边环 | 低透明（0.10）细描边同心环/多边 | 背景母题底纹，别用渐变 stroke（不保证，见技法库 D） |
| 母题结构 | 给页一个主结构（球/双环/hub六边/照片），不是卡片网格 | 每页选一个母题，见技法库 §B |

> **一句话**：华丽来自**层次 + 母题 + 克制立体**，不是堆装饰。凡看到"纯色块、零透明、零阴影"——那就是还没设计完。

## 四、出件方式：版式永远我定，每个元素四选一（2026-06-17 用户定调）

**版式 / 构图（§1 那套）永远是我自己设计的**——clone / 素材 / 烘焙只用来"填格子"，**不替我决定版式**。定好版式后，**每个元素**按下面四选一，遇到搞不定的别死磕：

| 出件方式 | 用在哪 | 怎么做 |
|---|---|---|
| ① **自己画（原生注入）** | 默认；绝大多数平面立体感件（卡片 / 同心层叠 / 渐变 / 阴影 / 通透 / 文字 / 正多边形） | python-pptx 原生构造，最可控 |
| ② **矢量素材 + 调格式** | 已蒸馏进库的复用母题（球 / 光晕 / 六边母题 / 胶囊…）、`templates/icons` 的图标 | 取 `../assets/<风格>/catalog.json` 的件 → 换主题色 / 改尺寸 / 换字；**素材在哪 / 矢量·图片·scene3d 各怎么调 → 见 [`资产地图.md`](资产地图.md)** |
| ③ **clone / 同文件搬运** | **我注入不了 / 把控不好的**：真 3D 旋转棱台、复杂多效签名件 | 复制模板该 `sp`/`grpSp` → 只回流文字 / 换色，原生效果原样保留；**`scene3d` 件在「已含该件的同一文件内」可直接复制 / 搬运 / 抽子素材而保 3D**（与「注入新文件丢 3D」两回事，2026-06-17 立柱合并实证） |
| ④ **烘焙成 PNG** | 引擎不可表达：渐变文字 / 弧形 textPath / 想要 3D 旋转又不克隆整页 | 渲成图用 `<image>` 放回 |

**怎么选**：能自己画好 → ①；库里有现成 → ②；**一处注入不了或我把控不好，别耗——那一小片 ③ clone 或 ④ 烘焙，精力留给版式和整体**。一页里 clone / 烘焙只占**局部签名件**，版式与其余仍是我设计 + 自画 / 素材。**别因一处做不好就让整页扁平**。

**注入能力底表**（决定某件归 ①②③④ 的依据）：

| 元素 | 能否 python-pptx 注入 | 归类 |
|---|---|---|
| 平面立体感件：正多边形 / 同心层叠 / 渐变 / 阴影 / 通透 / 描边 / duotone 图 / 胶囊 | ✅ 正常渲染（已验证） | ① 自己画 / ② 素材 |
| `scene3d`/`sp3d` 真 3D 旋转·棱台 | ❌ 注入**新文件**即被丢弃（验证多次）；✅ 但**同一文件内复制 / 搬运保 3D** | ③ clone / 同文件搬运（或 ④ 烘焙成带 3D 的图） |
| 渐变文字 / `textPath` 弧形字 | ❌ 渐变压纯色 / textPath 禁 | ④ 烘焙成 PNG |

> 现代审美要**平面立体感**（§1.0），真 3D 既过时又注入不了——"做不出 scene3d"基本不影响设计感；真要模板那种 3D 旋转倾斜，③ clone 或 ④ 烘焙，别死磕。

> **矢量图不是铁板一块**：一个 `grpSp`（组合）在 XML 里就是一组子节点，**任意子素材都能单独抽取 / 缩放 / 重组**——要"同一矢量图里的不同部件"不必拆成多份，进组取子节点即可。配合上面"同文件内复制保 3D"，**搞定可编辑 3D 不再只有 clone 整页一条路**：只要目标落在已含该 3D 件的同一文件里，脚本就能搬运 / 复制 / 拆装（2026-06-17：把模板 p19 大柱三联 + 迷你双柱合成一页"多尺寸立柱族"）。

## 五、生成后自检清单（这就是外部化的"审美评判器"·逐条问·任一 NO 回修）

> 人类设计师下手前那只"眼睛"，我用这张清单代替。生成完**先自己过一遍**，别等用户挑。

- □ **景深**：背景到主体有没有 ≥3 层？有没有半透明叠加？（全平 → 退回 §1.1 分层）
- □ **光影**：所有发光/投影同一方向、同一光色？投影 `≤0.20`、≤3 件浮起？
- □ **占版与字号**：主视觉 ≥~45%？没缩在中央留大白？字号按 §1.3 刻度（内容页标题/引导 28、正文≈14、签名页主标按模板实测放大）、不低于模板实测？
- □ **签名页主文字够大**：封面 / 目录 / 章节的**主标题按模板实测放大**（量 `sz`÷100，别套内容页的 28；如 p19 章节标题 = **66pt**、PART = 40pt），生成后核对别又缩水。
- □ **形:字比例**：复用胶囊 / 卡 / 编号等**文字类素材**后，字号按放置尺寸**重设过**了吗？（没设 = 沿用写死的小字 → 一放大就显小，§六 13）
- □ **文本↔容器适配** ：胶囊 / 卡 / 徽章里的文字，容器**够长够高、目标文案一行放得下**吗？（尤其**加粗 / 换文案后变长**）——宁可**容器加宽适配文字**，也别让文字折行 / 溢出 / 截断（§六 16）
- □ **无重叠 / 碰撞（🆕 以前漏这项 = 漏检根源）**：**逐对核对任意两元素**——相邻文本块之间、文本压图形，有没有**重叠 / 压字**？（p10 把"主文案"放大成 3 行压住正文就是漏了这项；改字号 / 移位后**必重渲逐对看**）
- □ **行距 / 段距统一**：**多行正文 / 描述段** `lnSpc` 跨页一致（统一 120%）、段距合理、**别忽松忽紧**？单行标题 / 标签 / 表格单元格、**图表精调数值标签**没被误套 lnSpc（套了会基线下移、破坏精调位）？（§1.3）
- □ **参考 ≠ 照抄、避免雷同**：版式是**按真实内容从 `页型配方` 蓝图 adapt** 的，不是逐字搬模板 / 搬另一页？同一 deck **不同页型没长成一模一样**？（过渡页等可有意复用，其余别千篇一律）
- □ **凸出**：有 1–2 件受控越界出血？还是全龟缩在框里？
- □ **平面立体感(非纯三维)**：深度是**平面层叠**(通透半透明 + 精调柔阴影 + 细描边)做的，**不是** glossy 立体球 / 强内发光那种过时纯三维？（§1.0）
- □ **非扁平**：还有没有纯色实底块？能用渐变 / 通透 / 精调阴影 / 描边升级（§2）？
- □ **内容归位**：序号 / 文字 / 徽章都在所属卡片 / 容器**内**？（凸出只留给装饰）
- □ **内容图形分量**：真内容（文字/数据）有没有自己的图形分量（卡 / 底板 / 容器）？还是裸浮、被装饰（母题/图标）压过？（内容 ≥ 装饰，§6.20）
- □ **尺寸统一且够大**：同类编号 / 图标尺寸**一致且不偏小**？（别跨版本越改越小——生成后逐个量）
- □ **hub 圆心填充**：环形/枢纽母题的**圆心图标够大、填住中心**？（别在大环里缩成小点留一圈空——p36 中心赞踩可更大，§3.5）
- □ **自检真落实**：上一版自己提的修改，这版**逐条改了并核对**了吗？（别只列不改）
- □ **母题**：这页有没有**一个**主结构母题（球/双环/hub六边/照片）？还是又退化成卡片网格？
- □ **元素沿母题几何分布** ：相关小元素（点/标签/节点）是否**沿母题的圆弧/六边/曲线**排布，而非默认纵列/网格？（沿几何更有设计感，§3.4）
- □ **圆内/非矩形容器文字**：文字按容器**真实弦宽**吃进去了吗？没溢出圆/菱形边界？（§六 16）
- □ **素材**：`catalog.json` 里有现成件的，用了没？（有就别手画）
- □ **主题色**：全页是否由 accent1/accent2 统一？有没有冒出杂色？
- □ **几何究因**：六边形等是**正多边形**(没被压扁)？要"倾斜立体"是另作 3D 旋转(clone-only)还是走平面层叠——**别做成 glossy 球**？
- □ **grpSp 内文字居中**：组合内的节点 / 编号 / 标签文字在容器**正中**？（grpSp 有缩放 / 平移时子件「组内坐标」≠画布坐标 → 别手算坐标叠 textbox，蒸馏时**保留文字 run** 让 PowerPoint 自己算，§6.22）
- □ **字体=微软雅黑（非宋体）**：所有文字 run 是否有显式 `ea=微软雅黑`？（靠主题字体 → 注入新 pptx 中文回退宋体；python-pptx `font.name` 只设 latin 不设 ea，§6.23）

## 六、必须规避的坑（每条都踩过，写下来不再踩）

1. **扁平化**：纯色块、零透明、零阴影、所有块同一种格式 → 见 §2 升级表。
2. **胆怯小字 + 大留白**：AI 手感缩字、元素缩中央 → 见 SKILL §纪律10、记忆 `ppt-readable-scale`。
3. **照抄模板版式当"我的设计"**：版式要按**真实内容**重组（容量匹配组件、回流），不是平移模板坐标。
4. **字面执行用户的话、不核对模板**："白边"事件（用户说"有白边"，我没看模板就全改纯白，其实模板是半透明/灰）→ **模板是 ground truth，照话改之前先自己看模板**。
5. **拿 scene3d 注入做真 3D**：必丢 → 真 3D 走 clone+reflow（§4）。
6. **素材堆太多想不起来用**：素材库**小而精**；catalog 标清"放哪·什么效果·怎么调"，生成时先查库再手画。
7. **纯三维 glossy 立体感(过时)**：volumetric 渐变球 / 强高光 / 强内发光 / 重棱台 = 2000s–2010s 旧风，现在显廉价 → 一律改**平面立体感**(§1.0)。
8. **功能内容凸出容器外**：序号 / 文字探出卡片 = 错；凸出只给装饰(§1.4)。
9. **抄表象不究因**：模板六边形"看着扁"其实是正六边形 + 3D 旋转；别把"扁"这个表象直接抄成被压扁的形(2026-06-17 用户纠)。
10. **编号 / 图标越改越小、尺寸不一**：功能件要统一且够大，生成后逐个核对(2026-06-17 用户纠；参 [[ppt-readable-scale]])。
11. **自检只列不改**：上一版给自己的意见，下一版必须真正落实并逐条核验，否则等于没自检(2026-06-17 用户纠)。
12. **赖 PNG**：`render_truth.ps1` 是 PowerPoint COM 真渲染 = 用户所见，**非有损导出**；看到弱 / 平就是设计本身，别赖导出。
13. **形:字比例失配 → 复用字变小（"字小"根因之一）**：带文字的素材（胶囊 / 卡 / 编号六边）字号是**写死的**，而 **OOXML 改形状框不会自动缩放字号**；把形状放大、字没跟着放 → 文字显小（如胶囊 251×86 里却塞 14pt）。**复用文字类素材必须按放置尺寸重设字号**（经验：框高 × ~0.4 ÷ 1.33 ≈ pt，或直接对齐模板实测）；存 / 预览也按素材**自然尺寸**摆，别盲目放大（2026-06-17 用户从胶囊 / 六边形看出）。
14. **存错素材**：只存"看得见、可注入、比例对"的件；**scene3d/sp3d 件**(注入即空)归 clone-only、别当普通矢量；**淡到看不见的**(低 alpha 柔边)单独存意义不大。目录中心真本体 = `bz-hex-pad-white`(白透层,可注入) + `bz-hex-3d-base`(3D 层叠,clone)。
15. **python-pptx 删页留孤儿 / 重名部件**：`slides._sldIdLst.remove()` 只摘 sldId，**物理 slide 部件 + rel 残留**，再 `save()` 易撞出重名 `slideN.xml`（PowerPoint 可能读错页）。删页要连 `drop_rel(rId)` + 删部件，或事后 **COM SaveAs 重存** / **zip 层删孤儿 + 去 presentation.rels 残引**修干净；改完务必 zip 复查"无重复部件"（2026-06-17 立柱合并踩过并修复）。
16. **文本不适配容器（长度）→ 折行 / 溢出（"PART TWO" 胶囊事故）**：**自己画**带文字的容器（胶囊 / 徽章 / 标签）时，容器宽度要**按文字长度定**，让目标文案**一行放下**；我把模板文案加粗了却没加宽胶囊 → 文字折成两行。**根因**：[[ppt-readable-scale]] 与本节 §13 只管了"字号 vs 框高"，**没管"文字长度 vs 框宽"**。**通则——任何文本进任何容器（自画的 / 复用素材的 / 模板的）都要做适配**：要么**容器随文字**（加宽 / 加高 / 自适应），要么**文字随容器**（SKILL 纪律5 的 `content_contract`：折行→缩字→扩块→换组件）。生成后**逐个文本核对：没折行 / 没溢出 / 没截断**（2026-06-18 用户从胶囊看出）。**非矩形容器（圆 / 菱形）更窄**：顶底收口，按容器真实弦宽收字、别按外接矩形（p25 圆标"传统人工运营"溢出圆边 = 小瑕疵）。
17. **蒸馏错对象（存了通用图标）**：矢量库只收**结构 / 母题**（双圆 `bz-venn-dual` / 六边 hub / 层叠）和**风格件**（渐变胶囊 / 光晕 / 天际线带）；**通用图标**（齿轮 / 机器人 / 箭头…）是 `templates/icons`（≈7981 个）的活，**别蒸馏进矢量库**。p25 我误存 2 个质检图标，已撤到 `templates/icons/_harvested`（2026-06-18 用户纠）。
18. **schemeClr 注入新文件 → 配色全错（p36 泛粉事故）**：素材若用 `<a:schemeClr val="accent2">` 等**主题色引用**，注入**全新 pptx** 时按的是**默认 Office 主题**（accent2=橙、accent1=蓝灰…）而非 business-blue → 颜色全跑（p36 亮底泛粉 = accent2 被当橙）。**注入前必把 schemeClr→srgb 归一**（accent1→0165FF / accent2→00BEFF / bg1→FFFFFF / tx1→0A2A55…），或给新文件套 business-blue 主题。蒸素材时就该一律 recolor 成 srgb（`bz-ring-radial` 已脱主题没事，`bz-light-bg` 原带 schemeClr 故踩坑、**2026-06-21 已 recolor 成 srgb 修复**，2026-06-18）。
19. **一个 spPr 两个 `effectLst` → PowerPoint 拒开（E_FAIL，python-pptx 不报）**：python-pptx `shape.shadow.inherit = False` 会塞一个**空 `<a:effectLst/>`**（阻断继承）；若再手写 `<a:effectLst>` 加 `outerShdw`，spPr 里就有**两个 effectLst**，违反 schema → **PowerPoint COM Open 直接 E_FAIL**（而 python-pptx / lxml 容忍、zip 也"合法"，骗过一切 → 只在真渲染暴露）。修：加阴影前先 `spPr.find('effectLst')` **复用已存在那个**，没有才新建。**通用排查**：渲染 E_FAIL 但 zip / python-pptx 都合法时，**分阶段累积快照二分**定位哪层元素让 PowerPoint 拒开（p36 即如此定位，2026-06-18）。
20. **内容图形分量不低于装饰（seg-4 文字裸浮事故）**：页面"母题/装饰"（六边、刻度盘…）有强图形，真**内容（文字）却裸浮无背景** → 视觉权重倒挂、装饰喧宾夺主。这是 §3.3 红线的延伸：不只是"指示要压过装饰"，而是"**内容的图形分量 ≥ 装饰**"。尤其本 skill 要做"**脱稿也能读**"的成品，文字是真内容。解：给内容一张**轻卡 / 底板** + 放大字号，让内容与装饰各有图形分量；母题可留（2026-06-20 用户纠 p45：图标有六边、文字却没背景）。
21. **装饰背景件必须有意义且避让标题/内容（p45 角落 ghost 六边事故）**：随手加的"氛围"装饰（淡 ghost 六边等）若**无明确构图作用**、又**压过标题/正文** = 减分。装饰要么服务母题/景深（§1.1）、要么不放；且**绝不与标题/内容重叠**（v1 角落 ghost 六边穿过页标题"痛点分析"，2026-06-20 用户纠）。
22. **grpSp 组变换致文字偏移（p30 节点偏移事故）**：`grpSp` 的 `off/ext ≠ chOff/chExt` 时组被**缩放 / 平移**，子件「组内坐标」**不等于**画布坐标（p30 闭环组放大 1.1851 倍 + 平移）。我蒸母题时**清空节点文字、再用子件组内 off 叠新 textbox** → 节点字偏 0.1~0.6 寸（用户一眼看出没在圆心）。**解（通则）**：「自带文字槽位的母题」（grpSp 内节点 / 编号 / 标签）**蒸馏时保留文字 run**（清水印、留 run 位置 + 字号 + 颜色），回填**只改 `a:t` 内容**——位置 / 字号 / 色由 PowerPoint 按组变换自动算，零偏移还省量字号。**能保留就别重算坐标**（2026-06-20 用户纠并授「保留文本为文本方便确认位置」）。亦见 §0 步4① 与 §5『grpSp 内文字居中』。
23. **文字注入新 pptx 回退宋体（字体没烤进 run · p63 用户纠）**：模板保留的文字 run 多**无显式字体**、靠主题 `minorFont`；注入到 python-pptx 新建 pptx（默认 Office 主题、中文 `ea` 空）→ 中文**全变宋体**（PPT 大忌）。**与 schemeClr 泛色（§6.18）同一类坑**——靠宿主主题就会跑。**解（通则）**：蒸 / 自画时把字体**烤进每个 run**——`<a:latin/ea/cs typeface="微软雅黑">`，**`ea` 管 CJK 字形、必设**。**坑中坑**：python-pptx `font.name` 只设 `latin` 不设 `ea` → 我自画的标题 / palette 标签当时也是宋体；修复器要按"`ea` 缺失 / 为空就补"触发（**非"有 latin 就跳"**），并保留可能的 Bebas 数字 `latin`。全库矢量 + gallery/master/palette/preview 已回烤校正（2026-06-20）。亦见 §0 步4① 与 §5『字体=微软雅黑』。
24. **蒸馏粒度：蒸骨架 / 风格 / 背景，别蒸内容（p63 用户纠"蒸过头 + 漏蒸背景"）**：母题只蒸**结构骨架 + 风格件**（路径 / 节点圆 / 枢纽 / 渐变 / **背景**），**内容**（图标 / 正文 / 副文本 / 阶段名）**别蒸进资产**——用时再加（图标从 `templates/icons` 叠白、文字另置）；把整页内容烤进资产 = 蒸过头、不可复用（p63 v1 把 7 图标 + 全文字都烤进去了）。**反向同样错**：有设计感的**背景别漏蒸 / 别用平底糊弄**——p63 那张 `duotone+alphaModFix` 速度模糊底（`bz-speed-blur-bg`）是亮点，我 v1 用平 `bz-light-bg` 替 = 损失。**判据**：这件**换主题 / 换内容还能用吗**？能 = 该蒸（骨架 / 背景 / 风格）；是这页专属文案 / 图标 = 别蒸、用时加。
25. **`<p:pic>` 用 `a:blipFill` → PowerPoint 拒开（E_FAIL，python-pptx 不报 · p98 用户纠）**：`p:pic` 的填充元素是 **`p:blipFill`**（presentationml 命名空间）、里面才是 `a:blip`/`a:stretch`；写成 **`a:blipFill`**（drawingml）时 lxml / python-pptx / zip 都容忍能存，但 **PowerPoint COM Open 直接 E_FAIL**（与 §6.18 schemeClr 泛色、§6.19 双 effectLst、§6.23 字体回退 同属"错命名空间 / 靠宿主、只在真渲染暴露"一类坑）。**解**：注入图片（手机填图 / 特性图标 / showcase 填图）一律 `<p:blipFill><a:blip r:embed=…/><a:stretch/></p:blipFill>`，或直接用 python-pptx `add_picture`（结构保证对）。p98 特性图标 + `bz-showcase` 填图均已修（2026-06-21）。**排查**：渲染 E_FAIL 但 zip / python 都合法时，二分定位哪元素让 PowerPoint 拒开（本次"波浪 OK / 图标 FAIL"即此法）。
