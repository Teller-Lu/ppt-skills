# 运行时使用流程(做一份 PPT 的实际路径)

> 回答:ppt-skills 建成后,我实际做 PPT 走什么路径、靠什么原理决定配色/字体/logo/画布/页结构、提供图片怎么处理。
> 前置:[设计方案.md](设计方案.md) · [实施计划.md](实施计划.md) · [ppt-master解析.md](ppt-master解析.md)。

## 核心原理(一句话)

ppt-master 每次靠**八项确认现场发明**设计;
**ppt-skills 把设计 DNA 预先锁在你的风格库里——你只要"选一套风格",配色/字体/logo/画布/页结构就全定了。**
这就是它更省心、更稳、更接近成品的根。

---

## 你的一次完整使用(7 步)

```
你: 给文档(或说要求) [+图片] [+选风格]
 │
 ├─1 内容入料     文档→Markdown 事实源;没文档→口述要点整理成大纲
 ├─2 选风格 ★     挑一套(商务蓝/技术深色/学术简约)→ 配色/字体/logo/画布/页结构一次锁定
 ├─3 规划每页     内容大纲→风格页型;为每页定 骨架+组件+节奏;图片定位
 ├─4 图片处理 ★   analyze 元数据→定放哪页/怎么排版/裁剪嵌入/必要时遮罩叠色
 ├─5 逐页生成     Executor 取骨架+组件→带上下文组装微调成 SVG(实时预览可标注)
 ├─6 双质量门     技术门(沿用)+ 美学门(我们加:溢出/碰撞/对比/对齐/字号)
 └─7 导出回看     → exports/*.pptx(原生可编辑+讲稿+动画);你再微调
```

### Step 1 · 内容入料
- **有文档**(PDF/Word/PPT/Markdown/网页)→ 自动转 Markdown 作事实源。
- **没文档**→ 你说清主题+要点+场合+受众,我整理成大纲;完全没素材可先做主题调研(可选/二期)。

### Step 2 · 选风格 ★(这一步定下你问的全部设计要素)

**你关心的每一项,分别来自 brand 还是 layout:**

| 你关心的 | 来自 | 内容 |
|---|---|---|
| 配色 | **brand** | 主色 / 辅助色 / 背景,锁成 HEX |
| 字体 | **brand** | 标题/正文字体族 + 字号体系 |
| logo | **brand** | 放封面 / 页脚(具体位置由 layout 定) |
| 语气 / 图标风格 | **brand** | voice + icon style |
| 画布(16:9 / 4:3 / 竖屏…) | **layout** | 比例 + 安全区 |
| 页结构 / 页型 | **layout** | 封面/目录/章节/要点/对比/图表/结尾的骨架 |

- 你**挑一套风格** = 一个 brand + 一组 layout → 上表**一次全锁定**。
- **不挑** → 给默认风格 + 让你一句话选;支持**局部覆盖**("用商务蓝,主色换成公司红 #C8102E""这次用 4:3""这次别放 logo")。
- **"它怎么知道"的根源**:这些 DNA 是**第一次拆解你的精美 pptx 模板时提炼、写进 `brand.md` / `layout` 的**——所以不是它猜,是**你的模板早就告诉它了**,你只是在复用一套已验证的设计。
- 产出:本次 `design_spec.md` + `spec_lock.md`(写死本次 HEX / 字体 / logo 路径 / 画布 / 每页用哪个骨架 / 每页节奏)。
- **轻量确认**(比 ppt-master 八项大幅缩减,因为风格已托底):主要确认 受众 / 页数 / 大纲 + 你要的覆盖,然后一路自动跑到底。

### Step 3 · 规划每页(Strategist 角色)
- 把内容大纲映射到风格页型,为每页指定:用哪个 **layout 骨架**、组装哪些 **component 区块**、**页面节奏**(anchor/dense/breathing)。
- 你提供了图片 → 在这一步决定**每张图放哪页、用什么版式**。

### Step 4 · 图片处理 ★(你提供图片时怎么排版/处理)
1. **读元数据,不读像素**:`analyze_images` 抽每张图的 宽高比 / EXIF 方向 / 主色调 / 主体内容 → 文本。
2. **据此决策**:① 宽高比 → 位置(全屏 hero / 侧栏 / 卡片);② 主色调 → 和风格调色板是否兼容(不兼容则加遮罩/叠色统一);③ 主体内容 → 放哪一页。
3. **版式**:用图文词表 `Primary 主结构 + Modifier 修饰层`(hero 背景+原生覆盖 / 侧边对比 / 图作画布+注解卡…),**鼓励复合避免"扁平 AI 感"**。
4. **工程处理**:裁剪到目标比例(防拉伸)→ 交付时把位图复制进 pptx 的 media,用 `srcRect` 表达裁剪 → 必要时遮罩/叠色让它融进你的风格。
5. 你的图 = `user` 来源:**不生成、不搜索,只负责摆放与处理**。

### Step 5 · 逐页生成(Executor + 我们的组件化 B)
- 自动起**实时预览**(localhost:5050),你能边看边标注。
- 逐页:重读 `spec_lock` → 取该页 layout 骨架 + 选定的 component 区块 → **带完整上下文组装并微调**成该页 SVG。
- 这不是从零画,也不是脚本批量生成,而是"用你的预制件拼+调"——更快、更稳、更像你的模板。

### Step 6 · 双质量门
- **技术门**(沿用 ppt-master):禁用特性 / viewBox / spec_lock 漂移 / 图表几何。
- **美学门**(我们加的 C):文字溢出 / 元素碰撞 / 对比度 / 安全边距 / 对齐栅格 / 字号下限,阈值按你的风格校准。
- 有 `error` → 定向重生成该页(无 auto-fix,保留设计意图)。

### Step 7 · 导出 & 回看
- `total_md_split`(讲稿)→ `finalize_svg`(嵌图标/图片)→ `svg_to_pptx` → `exports/*.pptx`(**原生可编辑** + 讲稿备注 + 默认动画)。
- 你在 PowerPoint 里直接改(原生形状),或在实时预览里标注 → 定向重生成。
- 因为走了"质量地板 + 组件 + 美学门",到你手里时应当**比 ppt-master 默认输出更接近成品,精装修更少**。

---

## 和 ppt-master 的关键差别(为什么更省心)

| 环节 | ppt-master | ppt-skills(我们) |
|---|---|---|
| 怎么定配色/字体/画布/页型 | 每次八项确认,AI 现场推荐 | **选一套风格,DNA 一次锁定**(可覆盖) |
| 设计质量来源 | AI 当场的品味(方差大) | **你已验证的模板(质量地板)** |
| 版式生成 | 逐页从零手写 | **组件组装 + 微调**(更快更稳) |
| 质量把关 | 只技术门 | 技术门 + **美学门** |

---

## 没有匹配风格时的兜底
风格库为空 / 没有合适的一套 → 回退到"自由设计 + 八项确认"(即 ppt-master 原味),或**先用一份 pptx 现场提炼一套风格**再做。

---

## 你做的 vs 系统替你做的
- **你**:给内容、选风格(或用默认)、(可选)给图、最后回看微调。
- **系统**:转料 → 锁 DNA → 规划每页 → 组件组装逐页生成 → 双质量门 → 导出可编辑 pptx。

---

## ⚙️ 确认力度 = 可选模式(已定)

三档**都做成可选模式**,你每次可一句话指定;不指定时默认 **A**。

| 模式 | 触发 | 行为 | 适用 |
|---|---|---|---|
| **A 轻量确认(默认)** | 不说 = 默认走 A | 给内容+选风格后,系统回一份"受众/页数/大纲/排版打算"让你点头或改,再一路跑到底 | 大多数情况 |
| **B 极简直跑** | "直接生成 / 极简模式" | 跳过确认,给内容+选风格就一路生成到底 | 赶时间、风格内容都很明确 |
| **C 完整确认** | "完整确认 / 八项确认" | 逐项过完整确认(≈ ppt-master 原味) | 正式汇报、强场景、要逐项把关 |

例:"用技术深色,**极简直跑**,15 页" / "学术简约,**完整确认**"。
