# 素材模型与组件库规格

> 回应你的澄清:素材分**三型**,且**按内容组织**。本文定义三型、提炼方式,并给出**组件库 v1 规格**(内容类型清单 + 元数据 schema + 组件粒度)。
> 这是 [设计方案.md](设计方案.md) §风格库 的展开,也是阶段 3/4 的施工规格。

## 一、三种素材类型(你的真实模板形态)

| 类型         | 一个 pptx 装什么                       | 风格          | 含整体框架? | 提炼产物 / 喂给                                     | 数量     | 本次示例? |
| ---------- | --------------------------------- | ----------- | ------ | --------------------------------------------- | ------ | ----- |
| **单风格型**   | 一套完整 PPT 的全部框架                    | 全篇**统一**    | ✅ 含    | 1 个 style(brand+layout)+ 一组风格一致组件 → `styles/` | 较少(待挑) | ✅ 是   |
| **单内容汇总型** | **单个内容类型**的多种风格(如"目录"汇总、"数据图表"汇总) | 同内容、**多风格** | ❌ 不含   | 一批某内容类型的 component → `components/`            | 多      | 否     |
| **汇总型**    | **所有框架** + 每个内容类型都收集了多种风格         | 全面、**多风格**  | ✅ 含    | 一大批 component(覆盖全内容)+ 可参考框架 → `components/`   | **最多** | 否     |

**结论**:
- **单风格型** → 喂 `styles/`(成套"质量地板",负责**整体一致性**:配色/字体/画布/页栅格)。
- **单内容汇总型 + 汇总型** → 喂 `components/`(按内容类型组织、覆盖多风格的"零件库",负责**版式变体**)。
- 风格 = 一致性锚;内容组件 = 变体来源。两者在运行时配合(见 §八)。

## 二、为什么这个区分让 skill 更强

汇总型 / 单内容汇总型 = **按内容类型组织、覆盖多种视觉风格的"零件仓库"**。生成某风格 deck、遇到难排的内容(表格/对比/逻辑图/甘特图/组织架构…)时,系统从仓库挑一个**版式合适、风格相容**的零件,**重着色到当前风格**再用——proven 版式越多,AI 即兴越少,质量越稳。这是组件化(B)的弹药库,也是你素材量最大的部分。

## 三、内容类型清单(`content_type`;采用你的真实清单,会随模板生长)

| 组 | content_type |

## 四、组件粒度:默认"一整页"(回应你的疑问)

- **整页框架** = 占满一整张幻灯片的组件(封面、目录、结尾、一个"三段内容"页…)。
- **页内子块** = 一页里的一块区域(页面有统一标题栏+页脚,中间嵌一个"表格"块)。
- **你的素材是按"页"收集的,所以默认:一个组件 = 一整页**,最简单直接。
- "页内子块"只有当你想把"一张表格"当成能塞进任意页面的小积木时才需要——**一期先不拆子块**,以后真有需要再说。

## 五、提炼流程(阶段 3 起)

> **0 · 提炼前净化(sanitize,全局必做)**:work-pc 上公司 **DLP 会给每个 pptx 逐页注入保密图章**(本例 = `image10000.png`,右上角固定 ~1190,0,90×45);模板自身也可能带装饰水印(本例 = 小字"方能演亦")。**这些绝不能进 floor / 组件 / skill**(用户明确要求)。提炼前按三类规则剔除:① 资产名/异常高 index(`imageNNNNN`);② 右上角固定小图位置;③ 重复占位文字。已对第一套执行(清除 24 页 × 图章 = 48 处引用 + 删除资产)。

1. **判型**:单风格型 / 单内容汇总型 / 汇总型(启发式:是否含整体框架 + 全篇风格是否一致;或你直接标注每个文件的类型)。
2. **单风格型** → 提 `brand` + `layout` + 风格一致组件入 `styles/`;并**顺便校准美学门、生长组件分类**。
3. **单内容汇总型 / 汇总型** → 逐页判 `content_type` + 打 `style_family` 标签,入 `components/`;不产 brand。
4. **本次第一套(单风格型)**:走第 2 条——边解析边把每页归类成一个组件,**用真实页型"生长"出 v1 内容清单,并量出美学门阈值**(落实你"在识别过程中提炼设计"的想法)。

## 六、元数据 schema(`components/catalog.json` 每条)

`content_type` 为**主索引**,`style_family` 为次级标签(贴合你"按内容组织"的素材)。

```json
{
  "id": "cmp_table_zebra_dark_01",
  "content_type": "table",
  "style_family": ["dark", "blue", "flat"],   // 次级:视觉族,跨风格匹配用
  "parent_style": "business-blue",             // 来自单风格型则填;汇总类为 null
  "source": "数据图表汇总.pptx#p12",
  "canvas": "16:9",
  "slots": [                                   // Executor 要填的内容槽
    {"name": "title", "type": "text"},
    {"name": "rows",  "type": "table", "cols": 4, "rows": "3-8"}
  ],
  "color_roles": {                             // 可重着色映射 → 接 spec_lock 语义角色
    "#1B2A4A": "bg", "#3B82F6": "primary", "#E5E7EB": "text"
  },
  "bbox": {"w": 1280, "h": 720},
  "tags": ["zebra", "header-row"],
  "notes": ""
}
```

**让组件跨风格复用的两个关键字段**:
- `slots` —— 决定"能装什么内容"(标题、表格行列数、分段数…)。
- `color_roles` —— 把组件色映射到 spec_lock 语义角色(primary/secondary/bg/text),决定"能不能重着色到别的风格"。

> **⚠️ 组件是"蒸馏"出来的,不是原样搬运**(首套导出复盘的关键结论):提炼时把每页**改写**成仅用引擎安全构造的干净 SVG;引擎无法表达的效果(**文字图片填充、嵌入图表**)**烘焙成图片**;并为每个 slot 增加 **`content_contract`**(`max_lines` / 字号范围 / **`content_box` 隐形边界** / `overflow` 回流策略)与 **`technique_map`**(技法→引擎安全写法)。填真实内容时按 `content_contract` **自动回流**(折行→缩字→扩块→换更密组件)。完整方法见 [保真与内容适配方法论.md](保真与内容适配方法论.md)。

## 七、目录结构

```
skills/ppt-skills/
  styles/                       # 单风格型 → 成套"质量地板"
    <style-id>/                 # 如 business-blue
      brand.md                  # 配色/字体/logo/语气/图标 + 间距栅格 + 字号体系
      layout.md                 # 画布/页结构/页型 → 引用哪些 component
      gate.calibration.json     # 从该风格量出的美学门阈值
      source/                   # 原始 pptx 转出的 svg 留档
  components/                   # 单内容汇总型 + 汇总型 → 跨风格内容零件库
    catalog.json                # 索引:每个组件的元数据(见 §六)
    <component-id>.svg          # 组件本体(带 slots 与可重着色映射)
```

## 八、风格与组件在运行时怎么配合

1. 你**选一套风格**(单风格型来的 floor)→ 锁定配色/字体/画布/页栅格(整体一致性)。
2. 每页按 `content_type` 去 `components/` 取候选 → 选 `style_family` 与当前 floor **最相容**的 → 用 `color_roles` **重着色**到 floor 的 spec_lock → 填 `slots` 内容。
3. floor 保证"看起来是一套",组件库保证"每种内容都有 proven 版式"。

> **所以你不急着挑很多单风格型**:第一套就够起步当 floor;真正的弹药是你那些按内容汇总的(单内容汇总型 + 汇总型)。等一期跑通,如果想要更多 floor,你再挑几套好的单风格型即可。
