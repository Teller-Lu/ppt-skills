# 美学质量门规格(v1 提案)

> 回应 point 2:把"看着像 AI 做的"变成**可测、可定位、可回修**的缺陷。这是**一版提案**,你再加你的想法,我们融合。
> 接在技术质量门(`svg_quality_checker`)之后,跑在 `svg_output/` 上。底图见 [ppt-master解析.md](ppt-master解析.md) §9。

## 一、设计原则

1. **程序化可测**:每条都能用脚本在 SVG 上算出来,不靠 AI 主观判断(区别于 ppt-master 的 visual-review)。
2. **按你的模板校准**:阈值不是通用标准,而是从你的风格量出来的(每风格一份 `gate.calibration.json`)——所以是"按你的审美把关"。
3. **沿用 ppt-master 严重性模型**:`error` 阻塞 → 定向重生成该页;`warning` 提示不阻塞;**无 auto-fix**(机械 patch 会丢设计意图)。
4. **画布归一**:阈值尽量用画布百分比表达;16:9 以 1280×720 为参考,其它画布按比例缩放。

## 二、v1 检查项与阈值

| 维度 | 怎么量(程序化) | v1 阈值 | 严重性 |
|---|---|---|---|
| **文字溢出** | 每个文本 bbox 是否超出其容器 / 安全区 | 超出容器 >2px = error;超出安全区 = error | error |
| **元素碰撞** | 内容组两两 bbox 相交面积(排除背景/装饰/标注覆盖层) | 相交 >15% 面积且未标记 overlay = error;5–15% = warning | error / warn |
| **对比度** | 文字色 vs 其背景 WCAG 对比度 | 正文 ≥4.5:1;大字(≥24px 或 粗体≥19px)≥3:1 | <下限 error;临界 warn |
| **安全边距** | 非出血内容是否进入边距带 | 边距 = 画布每边 4%(可按 layout 覆盖);越界 = error(出血元素豁免) | error |
| **对齐栅格** | 元素 左/右/中/上 坐标是否贴共享参考线 | 距参考线 2–12px 的"差一点对齐" = warning(贴上=好,差很多=可能故意) | warning |
| **字号下限** | 渲染字号是否低于地板 | 正文 ≥ 画布高的 1.9%(720→约14px);标注 ≥1.6% | <地板 error |
| **字号种类** | 一页内不同字号的数量 | >5 种 = warning | warning |
| **留白/密度** | 安全区内容填充率 | <30% 或 >78% = warning | warning |
| **调色板合规** | 每个 fill/stroke 是否属于 spec_lock 调色板(含容差) | 偏离 >ΔE 阈值 = warning | warning |

## 二·补 新增两项检查(来自首套导出复盘)

- **隐形边界溢出**:文本/块越过组件声明的 `content_box`(设计上的隐形内容边界,**不只是画布边界**)→ `error`。根治 p10/p15/p20 类"文字超出块/越界"问题。
- **构造兼容**:扫描 SVG 里"引擎不可靠表达"的构造(`<mask>`、组 opacity、文字图片填充、嵌入图表对象…)→ `error`,提示改写或烘焙成图。杜绝"模板里好看、导出就崩"。

> 完整推理见 [保真与内容适配方法论.md](保真与内容适配方法论.md)。

## 二·三 非朴素门槛（richness gate）—— v2 新增 · 已实现

> **来历**：用户指出"**只会挑刺的门造不出华丽**——它只能拒绝丑，不能生成美"。所以美学门不只查缺陷，还要**强制丰富度下限**：一页"太朴素"（裸色块、零图标、无母题）= `error`，和文字溢出一样拦下。正向标准从 PowerPoint 真渲染提炼（见 [技法库.md](技法库.md) §B 母题目录 / §A 真图标）。

**丰富度信号（程序化可测）**：

| 信号 | 怎么量 | 权重 |
|---|---|---|
| `bg_visual` | 全幅 rect 渐变填充，或 ≥85% 画布的 `<image>` | 2 |
| `decoration_layer` | 径向渐变氛围圆，或低透明描边的幽灵环/多边形 | 2 |
| `gradient_shapes` | 前景形状用渐变填充的个数 | ≤3 |
| `icons` | `<use data-icon>`（或已内联的真图标）个数 | ≤3 |
| `depth` | 被引用的 blur 投影 / 发光 filter | 1 |
| `accent_or_motif` | accent 细条 / 六边形 / 多圆环母题 | 1 |

**按页型的硬要求 + 分数下限**（校准期可调）：

| 页型 | 必备信号 | 分数下限 | 必须有真图标 |
|---|---|---|---|
| cover | bg_visual | 3 | 否 |
| section | bg_visual + decoration | 4 | 否 |
| table / references / closing | bg_visual + decoration | 4 | 否（文本密集，豁免） |
| content / comparison / hub / data / timeline / toc | bg_visual + decoration | 6 | **是** |

缺任一必备信号、或（需图标页型）缺图标、或分数 < 下限 → `TOO_PLAIN` error，并**列出缺什么**，可定向回修。

**实现**：`scripts/aesthetic_checker.py`（stdlib，零新依赖；读该风格 `gate.calibration.json`，故按模板校准）。同一脚本也实现 §二 中**可静态计算**的缺陷检查：字号下限 / 字号种类 / 调色板合规（豁免装饰与灰阶）/ 安全边距（**全幅出血元素豁免**）/ 禁用构造扫描（mask、组 opacity、rgba、image-opacity、textPath）/ 渐变文字预警（导出会被压成纯色）。安全：拒绝带 `<!DOCTYPE>`/`<!ENTITY>` 的 SVG（防 XXE / billion-laughs）。

**已验证（2026-06-13，9 组件）**：富化后的 `ad-section` **PASS**（richness 8/4）；旧的 8 个裸组件被准确判 `TOO_PLAIN` 并指出缺 `bg_visual`/`decoration`/`icons`；`ad-seg3`（9/6 分够）仅因用**数字圆占位**被判"缺真图标"——正是接下来要修的点。→ 门确实**永远不会放过朴素页**。

## 三、校准:在解析第一套模板时量出

解析第一套(单风格型)时,**顺便测量**:实际安全边距、字号体系(有哪几档、最小多少)、对齐栅格步长、调色板 → 写进该风格的 `gate.calibration.json`,**覆盖上面的通用 v1 默认**。
→ 所以美学门是**跟着你的模板长出来的**,不是套一把通用尺子。

## 四、输出

报告每页的 `error` / `warning` 列表(维度 + 位置 + 实测值 + 阈值),`error` 触发该页重生成。格式对齐 ppt-master `svg_quality_checker` 的输出风格,便于复用其报告管线。
