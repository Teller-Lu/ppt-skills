## 一、两个"该我拍板"的架构决策

### 决策 1:自包含(self-contained),不引用外部 ppt-master

把需要的引擎**复制进本 repo**,而不是运行时去引用本机的 `D:\Lu.Yao7\github\ppt-master`。

**理由**:skill 推到 GitHub 后要能**独立安装运行**;若引用本机绝对路径,换一台机器、或别人安装就直接失效。自包含才能"一处 clone,处处可用"。

### 决策 2:复制 + 裁剪 ppt-master 引擎,不从零重写

借用 ppt-master 现成的脚本,自己只写薄薄一层 `SKILL.md` + 风格库。

**理由**:最有价值也最难的部分是 `svg_to_pptx`(SVG → 可编辑 pptx 的转换器)和 `finalize_svg` 流水线,重写等于造轮子(YAGNI)。**个性化都集中在我们自己写的那薄薄一层**,引擎拿来即用。

> 这两条若你不认可,可在评审时推翻;否则按此执行。

## 二、三层结构(沿用 ppt-master 的解耦思想)

| 层 | 内容 | 谁写 |
|---|---|---|
| **大脑** | `SKILL.md`:触发词、风格选择菜单、裁剪后的流程编排 | 我们自己写 |
| **肌肉** | `scripts/`:从 ppt-master 复制裁剪的引擎 | 复制 + 裁剪 |
| **审美** | `styles/`(单风格型→成套 floor)+ `components/`(单内容汇总型 / 汇总型→跨风格组件库),从 `.pptx` 提炼并净化 | 我们建设 |

## 三、要复制进来的引擎(裁剪后)

**保留**:
- `project_manager.py` — 建项目结构
- `source_to_md/` — 文档→markdown(按需)
- `svg_editor/` — 实时预览(flask,localhost)
- `svg_quality_checker.py` — SVG 质检
- `finalize_svg.py` + `svg_finalize/` — 嵌图标/图片、裁剪、文本扁平化
- `svg_to_pptx.py` + `svg_to_pptx/` — **核心:SVG → 可编辑 pptx**
- `pptx_to_svg/` — 把我的模板 `.pptx` 吃进来 → SVG(用于提炼风格)
- `total_md_split.py` — 讲者备注拆分

`references/` **保留**(设计规则/角色定义):`strategist`、`executor-base`、`shared-standards`、`canvas-formats`、`image-layout-spec` 等。

**裁掉**:
- `image_gen.py` + `image_backends/`(AI 生图)
- `notes_to_audio.py` + 旁白/narration
- 动画(`pptx_animations.py` 等,可选,先不要)
- `image-palettes/`、`image-renderings/`(若不生图)

## 四、依赖(本机 Python 3.10.1 ✓)

裁剪后**必装**:

| 用途 | 包 |
|---|---|
| 核心导出 SVG→pptx | `python-pptx`,`svglib` + `reportlab`(或改用 `cairosvg`) |
| 实时预览 | `flask` |
| 图片处理(finalize 用) | `Pillow`,`numpy` |
| 文档输入(按需) | `PyMuPDF`(PDF)/ `mammoth`+`markdownify`(Word/HTML)/ `openpyxl`(Excel)/ `requests`+`beautifulsoup4`(网页) |

**可不装**:`edge-tts`(配音)、`google-genai`(生图)、`curl_cffi`(可选反爬)。

## 五、风格库结构(styles/)

每套风格一个文件夹,互相独立:

```
styles/
  <风格名>/                 # 如 business-blue / tech-dark / academic-clean
    brand.md               # 审美 DNA:主色/辅助色/字体/留白/标题风格/调性/voice
    layouts/               # 招牌页型(SVG 骨架)
      cover.svg  toc.svg  section.svg  content.svg  chart.svg  closing.svg
    charts/  icons/        # 可选
    source/                # 提炼来源(原始 pptx 转出的 svg,留档)
```

> **⚠️ 已升级**:上面是"单风格型"的成套提炼;素材其实分**三型**(单风格型 / 单内容汇总型 / 汇总型),组件库是**跨风格**的,且提炼前要**净化**(剔除 DLP 保密图章 + 装饰水印)。修订后的 `styles/` + `components/` 结构、组件元数据 schema、判型与净化流程,见 [素材模型与组件库规格.md](素材模型与组件库规格.md)。
