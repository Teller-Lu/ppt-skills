## A. 真实图标（v2 核心新增）——别再用数字圆

图标库已进 `skills/ppt-skills/templates/icons/`：`tabler-outline`(5039，**本模板主力·白线**) / `tabler-filled`(1053) / `chunk-filled`(640) / `phosphor-duotone`(1248)。

**用法**（`finalize_svg` 导出前自动把 `<use>` 内联成真实 `<path>`，导出时无 `<use>` 残留 → 引擎安全）：
```xml
<use data-icon="tabler-outline/world" x="248" y="258" width="64" height="64" fill="#FFFFFF"/>
```
- `data-icon = 库/名`（文件名去 `.svg`）；`fill` 决定颜色；建议尺寸 32–64。
- **找图标**（零 token）：`ls templates/icons/tabler-outline/ | grep 关键词`。

**本模板已核对的图标语汇**（白线图标，置于渐变圆/六边形内）：

| 语义 | tabler-outline 名 | 语义 | tabler-outline 名 |
|---|---|---|---|
| 水系/治理 | `droplet` | 电力 | `bolt` |
| 网格化 | `grid-dots` / `grid-pattern` | 网络/拓扑 | `network` / `topology-star` |
| 分配/责任人 | `users-group` / `share-3` | 芯片/算力 | `cpu`（无 `chip`） |
| 监测 | `radar-2` / `activity` | 方法/工具 | `tools` / `settings` |
| 全球/区域 | `world` / `map-2` | 指标/目标 | `chart-dots` / `target` |

**图标徽章配方**（真图里反复出现：渐变圆 + 居中白线图标）：
```xml
<defs>
  <radialGradient id="badge" cx="0.5" cy="0.4" r="0.7">
    <stop offset="0" stop-color="#1C7FD0"/><stop offset="1" stop-color="#14319C"/>
  </radialGradient>
</defs>
<circle cx="280" cy="290" r="44" fill="url(#badge)"/>
<use data-icon="tabler-outline/droplet" x="252" y="262" width="56" height="56" fill="#FFFFFF"/>
```

## B. 母题目录（v2 核心新增）——一种页型一个母题

> 母题是"设计本身"，不是装饰。下面每个母题给：**来源页 → 骨架 → 引擎安全配方 → 内容槽**。
> （B1–B4 来自已深读的 4 页；B5+ 看更多真图后续补。）

### B1 无穷双环（来源 slide_15 全域治理）
**骨架**：两个相切大圆（渐变环）→ 环内放[图标徽章 + 2 行标题] → 左右各 3 个侧标签，用**放射小圆点 + 虚线**连到环 → 底部 3 个渐变胶囊钮。
**环 = 两盘叠**（外渐变盘 + 内 bg 盘 knockout，**不用渐变描边**）：
```xml
<circle cx="470" cy="300" r="150" fill="url(#ringGrad)"/>   <!-- 外:渐变盘 -->
<circle cx="470" cy="300" r="112" fill="url(#adBg)"/>        <!-- 内:bg盘,挖空成环 -->
<!-- 第二个环 cx 右移 ~250 使两环相切交叠 -->
```
**侧标签连接**（T7）：`<line ... stroke-dasharray="2 6" stroke-opacity="0.5"/>` + 端点 `<circle r="4"/>`。
**底部胶囊钮**（T9）：见下。
**槽**：`ring1_icon / ring1_title(≤6字×2行) / ring2_* / side_label_1..6 / pill_1..3`。

### B2 双圆对比 / 转化（来源 slide_18 研究综述）
**骨架**：左**深蓝盘**（图标）──三角箭头──> 右**亮蓝盘**（图标）；左右两栏正文；**双色标题**；背景**幽灵同心环**。
```xml
<circle cx="400" cy="300" r="70" fill="url(#deepDisc)"/>
<use data-icon="tabler-outline/file-text" x="372" y="272" width="56" height="56" fill="#FFFFFF"/>
<line x1="480" y1="300" x2="560" y2="300" stroke="#146DB4" stroke-width="4" marker-end="url(#tri)"/>
<circle cx="640" cy="300" r="70" fill="url(#brightDisc)"/>
<!-- marker：fill 必须 = line 的 stroke 色（#146DB4），三角 3 顶点，依 §1.1 -->
```
**幽灵同心环**：见 T2b。**槽**：`heading_a/heading_b（双色）/ left_para×N / right_para×N / disc_a_icon / disc_b_icon`。

### B3 中心 hub + 卫星六边形（来源 slide_19 配电网）
**骨架**：中心大圆角块（渐变）含主题词 → 4–6 个**六边形图标徽章**环绕 → 每个六边形对应一角文字块；背景**幽灵六边描边**。
**六边形**（`<polygon>`→custGeom，§1.2）：
```xml
<polygon points="250,210 294,235 294,285 250,310 206,285 206,235"
         fill="url(#badge)"/>
<use data-icon="tabler-outline/bolt" x="226" y="234" width="48" height="48" fill="#FFFFFF"/>
```
**槽**：`hub_title / hex_1..N(icon) / corner_text_1..N(标题+2行)`。

### B4 整幅照片封面（来源 slide_01）
**骨架**：全幅 `<image>` 照片 → 底部**蓝缎带**（渐变 `<path>`）→ 标题（纯色或烘焙渐变）→ 3 个 footer 图标项。
```xml
<image href="cover.assets/bg.jpg" x="0" y="0" width="1280" height="720" preserveAspectRatio="xMidYMid slice"/>
<!-- 标题可读性:顶部/底部加渐变遮罩 rect(§6),不要纯黑铺满 -->

<path d="M0,560 C420,500 860,640 1280,540 L1280,720 L0,720 Z" fill="url(#ribbon)"/>
```
**坑**：真图顶部那条**弧形点线校名文字** = `textPath`，**禁**（§2）→ **烘焙成 PNG** 或省略。**槽**：`bg_image / title / subtitle_en / footer_1..3(icon+label)`。

## C. 技法清单（修订 T1–T10）

**T1 背景微渐变**（别用纯白）
```xml
<linearGradient id="bg" x1="0" y1="0" x2="0.35" y2="1">
  <stop offset="0" stop-color="#F7FBFE"/><stop offset="1" stop-color="#EAF4FD"/>
</linearGradient>
```
**T2 柔光氛围圆**（半径 300–430，半出血，软）：`<radialGradient>` 中心 0.18–0.20 → 边缘 0。
**T2b 幽灵描边环 / 多边形**（**新**，真图的"细环底纹"，≠T2 的柔光盘）：
```xml
<circle cx="640" cy="300" r="240" fill="none" stroke="#146DB4" stroke-opacity="0.10" stroke-width="1.5"/>
```
（`stroke-opacity` 引擎安全，§2 line155/177；可叠 2–3 个不同半径成同心环。）
**T3 形状渐变填充**（卡/圆/六边都渐变，不用纯色）：`fill="url(#grad)"`。
**T4 品牌蓝投影**（**修订**：`flood-opacity` 收到 **0.12–0.20**，不是 0.32！依 §5；仅浮起/CTA 元素；单一光源 `dy` 同向）
```xml
<filter id="sh" x="-25%" y="-25%" width="150%" height="150%">
  <feGaussianBlur in="SourceAlpha" stdDeviation="10"/>
  <feOffset dx="0" dy="8" result="o"/>
  <feFlood flood-color="#146DB4" flood-opacity="0.18"/>
  <feComposite in2="o" operator="in" result="s"/>
  <feMerge><feMergeNode in="s"/><feMergeNode in="SourceGraphic"/></feMerge>
</filter>
```
**T5 渐变 accent 条**（标题/章节的渐变竖条/横条）：`<rect rx="3" fill="url(#acc)"/>`。
**T6 图标徽章**（**修订**：白线**真图标** `data-icon`，不是数字）——见 §A 配方。
**T7 放射连接**（**新**）：虚线 `<line stroke-dasharray="2 6" stroke-opacity="0.5"/>` + 端点 `<circle r="4">`。
**T8 箭头**（**修订**）：连接线用 `marker-end` 三角（marker `fill` **必须 = line 的 stroke 色**，宽高 3–15，§1.1）；块状箭头用独立 `<polygon>`。
**T9 渐变胶囊钮**（**新**，slide_15 底部那排）：
```xml
<rect x="120" y="600" width="280" height="64" rx="32" fill="url(#acc)" filter="url(#sh)"/>
<circle cx="160" cy="632" r="22" fill="#FFFFFF"/>
<use data-icon="tabler-outline/checklist" x="146" y="618" width="28" height="28" fill="#146DB4"/>
<text x="196" y="641" font-size="22" font-weight="bold" fill="#FFFFFF">丰富检验方法</text>
```
**T10 双色标题**（**新**，slide_18 的"研究 / 不足之处"）：相邻 `<text>` 或 `<tspan fill>` 两色（主蓝 `#146DB4` / 墨 `#222A35`），引擎按 run 上色。

## D. 引擎安全 & 坑（修订，标依据）

**✅ 已确认安全**（`shared-standards.md`）
- `stroke-opacity` / `fill-opacity` 做透明（§2 line155/177）——幽灵环、半透明叠层全靠它。
- `<polygon>` / `<path>` → `custGeom`（§1.2 line120）——六边形/菱形/异形。
- `marker-end` 三角/菱形/圆箭头（**条件**：marker 形是 3 顶点三角 / 4 顶点菱形 / circle，marker `fill` = line `stroke` 色，宽高 3–15；§1.1）。
- `stroke-dasharray` → `prstDash`（§7）——虚线/点线连接。
- `clipPath` **仅** 用在 `<image>`（§1.2）——可把照片裁成圆/圆角/六边。
- 图片遮罩用**叠加渐变 `<rect>`**（§6），不用 `<mask>`。

**⚠️ 不确定 / 绕开**
- **渐变描边**（`stroke="url(#grad)"`）不保证 → 环/圈用"**外渐变盘 + 内 bg 盘**"叠出（见 B1），别用渐变 stroke。
- `<image>` 上别直接挂 filter（图片阴影不一定转）。

**⛔ 禁用 → 替代**
- **渐变文字**（`<text fill="url(#grad)">`）导出**被压成纯色** → 强调字用**纯色**，非要花式就**烘焙成 PNG**。
- **弧形/沿路径文字** `textPath` **禁**（§2）→ 烘焙成 PNG（封面校名弧就是这种）。
- `<mask>` / 组 `opacity` / `rgba()` / `<image opacity>` **禁**（§2 line158）→ 用 `fill-opacity`/`stroke-opacity`/叠加 rect。

**关于"四选一"**：ppt-master 通则是"每容器只取 shadow/border/gradient/tint 之一，叠 = 模板感"。但本技能**以模板为质量地板**，真图本身就叠（渐变填充 + 轻投影），所以**允许按模板适度叠**——前提是把投影压到 `flood-opacity ≤0.20`、单一光源。

## E. 用法纪律（修订）

一页之内：
- **背景至多三层**：微渐变(T1) + 1–2 个柔光圆(T2) + 幽灵描边(T2b)。
- **一个主母题**（B1–B4 选一），不要两个母题打架。
- **浮起元素 ≤2–3 个**加投影，单一光源（所有 `dy` 同向）。
- **图标统一一套**（本模板 `tabler-outline` 白线为主），尺寸/线宽一致。

> 华丽 = 母题 + 真图标 + 层次，**克制堆叠**。
